Which metric is independent of the chosen threshold for classification?

📖 Ch 3 · Evaluation

Answer A · AUC
  1. AUCTRUE
  2. AccuracyFALSE
  3. PrecisionFALSE
  4. Balanced accuracyFALSE
Why AUC integrates ranking performance over every threshold — equivalently, it is the probability a random positive is scored above a random negative — so it never depends on a single cutoff. Accuracy, precision and balanced accuracy are all read off a confusion matrix, which only exists after you convert scores to hard labels at a fixed threshold.

Which of these four statements (about Spark) is correct / incorrect?

📖 Ch 8 · Big Data

Answer All four are true statements
  1. DStream as a continuous RDD.TRUE
  2. DataFrame = untyped Dataset[Row].TRUE
  3. Spark replaces map-reduce by building a directed acyclic graph (DAG).TRUE
  4. Spark is much more efficient on RDDs because it does more compute in memory.TRUE
Why All four are accurate: a DStream is literally a discretised sequence of RDDs (one per micro-batch); a DataFrame is an untyped Dataset[Row]; Spark’s engine schedules work as a DAG of stages with lazy evaluation; and its headline advantage over Hadoop MapReduce is in-memory computation that avoids disk I/O between stages.
Framing note: since every option is factually true, the “incorrect” version of this question would have a distorted option that the recollection lost. As transcribed, treat all four as correct statements.

Which of these statements is incorrect?

📖 Ch 3 · Evaluation & Ch 4 · Trees & Ensembles

Answer C · None are incorrect
  1. Train-test split needs to be done even before preprocessing.TRUE
  2. Random forest is robust to outliers.TRUE

Which of these statements (about weights of evidence, WoE) is correct / incorrect?

📖 Ch 2 · Preprocessing

Incorrect statement A
  1. WoE encodes a monotonic relationship with the target variable.FALSE
  2. WoE encoding is possible for a binary classification target.TRUE
  3. WoE vs. the information value (IV) of those variables.TRUE
  4. None of the above answers are correct / incorrect.FALSE
Why WoE replaces a bin/category with ln(%good / %bad). It does not inherently create a monotonic relationship with the target — monotonicity is an optional design goal achieved through deliberate coarse binning, so (A) is the false claim. WoE is defined for a binary target (its canonical credit-scoring use), and IV is built directly from WoE via IV = Σ(%good − %bad)·WoE, so (B) and (C) are true.

Which statement about regularization terms is correct?

📖 Ch 2 · Preprocessing

Answer B
  1. L1 (Ridge) adds a penalty ∝ the square of the coefficients.FALSE
  2. L2 (Ridge) adds a penalty ∝ the square of the coefficients.TRUE
  3. L1 (Lasso) adds a penalty ∝ the square of the coefficients.FALSE
  4. L2 (Lasso) adds a penalty ∝ the square of the coefficients.FALSE
Why L2 = Ridge penalises the sum of squared coefficients (λΣβ²) → smooth shrinkage. L1 = Lasso penalises the sum of absolute values (λΣ|β|) → sparsity. Only (B) pairs the right name with the right penalty form; the others mislabel L1↔Ridge or attach the square penalty to Lasso.

Which of these statements (interpretability / metrics) is correct / incorrect?

📖 Ch 3 · Evaluation

Incorrect statement C
  1. AU(RO)C vs. AUREC (regression error characteristic) for regression tasks.TRUE
  2. Permutation feature importance breaks the correlation between variables.TRUE
  3. Partial dependence plots are unable to model interactions between variables.FALSE
  4. An ICE plot is similar to a univariate partial dependence plot.TRUE
Why (C) is the false statement: a 1-D PDP shows only an averaged marginal effect, but a 2-D (two-way) PDP can visualise pairwise interactions — so the blanket “unable” is wrong. AUROC (classification) has the REC/AUREC analogue for regression (A ✓); permutation importance does break correlations by shuffling a column (B ✓ — the “doesn’t break” phrasing would be false); and a PDP is exactly the average of the per-instance ICE curves, so they are closely related (D ✓).

Which statement regarding the bias–variance trade-off is correct?

📖 Ch 4 · Trees & Ensembles

Answer C
  1. Bagging increases variance but not necessarily bias.FALSE
  2. Boosting increases variance but not necessarily bias.FALSE
  3. Bagging decreases variance but not necessarily bias.TRUE
  4. Boosting decreases variance but not necessarily bias.FALSE
Why Bagging averages many deep, low-bias / high-variance trees trained on bootstrap samples → it slashes variance while leaving bias roughly unchanged. Boosting works the opposite way: it sequentially fits the residual errors of weak learners, mainly cutting bias (and often raising variance). So (C) is correct and (D) is its mirror-image error.

Which deep-learning technique doesn’t help to prevent overfitting?

📖 Ch 5 · Deep Learning: Images

Answer C · Smaller batch size
  1. Batch normalizationhelps
  2. Dropouthelps
  3. Smaller batch sizedoesn’t
  4. Regularizationhelps
Why Dropout (kills co-adaptation), L1/L2 regularization (weight decay constrains capacity) and batch normalization (mild noise from per-batch statistics) are all deliberate overfitting controls. A smaller batch size is fundamentally an optimisation / memory choice — its gradient noise can incidentally aid generalisation, but it is not a regularisation technique, so it’s the odd one out the question is after.

Which is the map-reduce pseudo-code to get the list [4, 9, 25]?

📖 Ch 8 · Big Data

Answer A
  1. [2,3,5].map(lambda x: x*x)TRUE
  2. [2,3,5].reduce(lambda x: 2*x)FALSE
  3. [2,3,5].map(lambda x: 2*x)FALSE

Why map transforms each element independently and preserves the list: 2²=4, 3²=9, 5²=25 → [4,9,25]. reduce folds the whole collection into a single aggregate, so both reduce options fail on structure alone; map(2*x) would give [4,6,10].

Which statement about text mining is incorrect?

📖 Ch 7 · Deep Learning: Text

Answer A · the incorrect one
  1. TF-IDF scores high for words frequent in a document and frequent in the corpus.FALSE
  2. Bag of words encodes per-word frequency/binary and scales poorly.TRUE
  3. CBOW uses context words as input to predict a target word.TRUE

Why (A) inverts the IDF logic. TF-IDF = term-frequency × inverse document-frequency (log N/df), which down-weights words common across the corpus. A high TF-IDF needs a word frequent in one document yet rare across the corpus — not “frequent in the corpus.” The other three are accurate (BoW is high-dimensional/sparse; CBOW predicts the centre word from context; embeddings can be clustered into topics).

  2. Decision trees cannot deal with continuous variables.FALSE
  3. Decision trees cannot deal with missing values.FALSE

Which statement about working with class imbalance is correct / incorrect?

📖 Ch 3 · Evaluation

Incorrect statement A
  1. Undersampling removes highly similar majority-class observations.FALSE
  2. Oversampling adds duplicates of minority-class observations.TRUE
  3. Under/oversampling results in a biased prediction distribution.TRUE
  4. Never under/oversample the test set.TRUE
Why (A) is false as a general claim: random undersampling drops majority points at random, not selectively by similarity — removing “highly similar” points is the behaviour of informed methods (Tomek links, ENN). Plain random oversampling does duplicate minority instances (B ✓ — SMOTE instead synthesises); resampling shifts the class prior so predicted probabilities are biased unless recalibrated (C ✓); and you must keep the test set at its natural rate for honest estimates (D ✓).

📖 Ch 5 · Deep Learning: Images

Answer B · 8×8×4
  1. 8×8×3FALSE
  2. 8×8×4TRUE
  3. 10×10×4FALSE
  4. None of the aboveFALSE
Why Spatial size with no padding: (10 − 3)/1 + 1 = 8 in each dimension. Each 3×3 kernel automatically spans all 3 input channels and outputs a single 2-D feature map, so the output depth equals the number of kernels (4), not the channel count. Result: 8×8×4. Open question — Variational auto-encoders

40%

📖 Ch 5 · Deep Learning: Images

Part a

Architecture of a variational auto-encoder (VAE)

A VAE is a generative auto-encoder that learns a probabilistic latent space. Where a plain auto-encoder maps an input to a single fixed code, the VAE encoder maps each input to a distribution over latent space, and we sample from that distribution before decoding. This forces the latent space to be smooth and continuous, so decoding random samples yields realistic new outputs.

1 · Encoder (recognition network)

A neural network (dense for tabular data, convolutional for images) compresses the input x. Its output is not a code but the parameters of a Gaussian over the latent variable: a mean vector μ and a spread. In practice we predict the log-variance log σ² rather than σ, because it is unconstrained, numerically stable, and guarantees a positive variance after σ = exp(0.5·log σ²). This defines the approximate posterior q(z|x) = N(μ(x), σ(x)²·I).

2 · Latent sampling & the reparameterization trick

We must draw z ∼ N(μ, σ²), but you cannot backpropagate through a random node. The reparameterization trick moves the randomness outside the differentiable path:

z = μ + σ · ε,     ε ~ N(0, I)

Now μ and σ flow deterministically from the encoder while ε supplies the noise, so the whole network trains end-to-end by ordinary backprop.

3 · Decoder (generative network)

A second network maps the sampled z back to a reconstruction x̂ (e.g. pixels via sigmoid), defining p(x|z). After training you can discard the encoder and generate new data by sampling z ∼ N(0, I) and decoding it.

4 · Loss — the evidence lower bound (ELBO)

L = Reconstruction(x, x̂)  +  KL( q(z|x) ∥ N(0, I) )

The reconstruction term (binary cross-entropy or MSE) makes x̂ resemble x. The KL term pulls the encoded distribution toward a standard-normal prior, regularising the latent space so it stays centred, compact and continuous. For two Gaussians it has a closed form:

KL = −½ · Σj ( 1 + log σj² − μj² − σj² )

The two terms pull against each other — reconstruction wants to spread codes apart to store detail; the KL wants to collapse them onto the prior. Their balance yields a well-structured generative latent space. (The β-VAE variant scales the KL term by β to tune disentanglement.)

Key points
  • Encoder outputs μ and log σ² (a distribution, not a point): q(z|x)=N(μ, σ²I).
  • Predict log-variance for stability and guaranteed positivity.
  • Reparameterization: z = μ + σ·ε, ε∼N(0,I) — keeps the network differentiable.
  • Decoder p(x|z) reconstructs; generate by sampling z∼N(0,I).
  • Loss = reconstruction + KL; the KL regularises the latent space toward the prior.
Part b

Advantage of a VAE over a normal auto-encoder

The core advantage is a continuous, smooth, regularised latent space. Because the encoder outputs a distribution and the KL term pulls every encoding toward a standard-normal prior, the latent space is densely and smoothly packed — without the “holes” a plain auto-encoder leaves between training points. Concretely:

  • It is generative. Sample a random vector from the prior N(0, I), decode it, and obtain a plausible new data point. A vanilla AE cannot do this reliably — arbitrary latent points fall into undefined gaps and decode to garbage.
  • Smooth interpolation. Nearby latent points decode to semantically similar outputs, so interpolating between two encodings produces a meaningful morph.
  • More organised / disentangled representation. The prior constraint encourages structured, often more interpretable latent dimensions.

Trade-off: the KL regularisation and the stochastic sampling constrain the encoding, so VAE reconstructions are typically slightly blurrier / less sharp than a deterministic auto-encoder’s, which optimises reconstruction alone.
