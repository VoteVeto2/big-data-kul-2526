Partial Dependence Plots (PDP) and Individual Conditional Expectation (ICE) plots are two related model-agnostic interpretability techniques.

Ch. 03 · Evaluation

a
Explain how a PDP is constructed and what a single point on the curve represents.
Model answer
  • A PDP shows how the model's prediction depends on one (or two) feature(s) of interest, holding the others fixed in aggregate.
  • Construction: for each value of the feature under study, set that feature to the value in every observed row while keeping each row's other feature values as they are, let the trained model predict on all rows, and average the predictions. (A coarser variant instead fixes the other features at a single median/mode reference.)
  • A single point on the PDP is therefore the AVERAGE model prediction over the dataset for that fixed value of the feature, sweeping the feature across its observed range.
b
Explain how ICE differs from PDP and one situation where ICE is more informative.
Model answer
  • ICE uses the same idea of varying the feature over a grid, but plots ONE line per original instance instead of a single averaged line.
  • For each instance, new rows are generated across the feature's grid and predicted; this yields one curve per instance (the PDP is essentially the average of all ICE lines).
  • ICE is more informative when there is heterogeneity across instances: the averaging in a PDP can hide that the feature pushes some instances up and others down. ICE reveals such diverging behavior (e.g. interaction-driven effects) that a flat or monotone PDP would mask.
c
Both PDP and ICE hold other features fixed (ceteris paribus). Explain the resulting pitfall and why one should inspect both the raw data and the PDP.
Model answer
  • Ceteris paribus means interaction effects can be invisible: e.g. with y ~ x0·x1 + x2 + noise, the PDPs of x0, x1, x2 may all look like flat or simple increasing lines because everything else is held fixed — absence of evidence is not evidence of absence.
  • A second, related pitfall: because the feature of interest is varied independently of the others, PDP/ICE can probe the model at unrealistic feature combinations when that feature is correlated with the rest (out-of-distribution points), which can distort the curve.
  • Conversely, looking only at the raw data (e.g. binned feature vs % positive) can suggest correlations that are not real once interactions are accounted for.
  • Example: the data may show 'sales drop for customers aged 30–40' while the PDP says the effect of age is constant — the mismatch signals an interaction, meaning age alone is not a sufficient explanation.
  • Hence one should inspect BOTH the raw data and the PDP/ICE to distinguish genuine effects from interaction artifacts.

Variational autoencoders (VAEs) build on ordinary autoencoders for generative tasks.

Ch. 05 · Deep Learning · Images

a
EXPLAIN how the encoding step of a VAE differs from that of a standard autoencoder, and state the two terms that make up the VAE loss function.
Model answer
  • A standard autoencoder maps an input to a single point in latent space: \(z = e(x)\).
  • A VAE instead encodes the input as a probability distribution over latent space, \(p(z|x)\): the encoder outputs the mean \(\mu_x\) and the (co)variance \(\sigma_x\) of a Gaussian. A point \(z\) is then sampled from this distribution and decoded.
  • The VAE loss has two terms: a reconstruction term \(\lVert x - \hat{x} \rVert^2\) plus a KL-divergence regularization term \(\mathrm{KL}[\,N(\mu_x, \sigma_x),\, N(0, I)\,]\) that pushes the encoded distributions toward a standard normal.
b
EXPLAIN why the KL regularization term is necessary and what would happen to the latent space without it.
Model answer
  • The KL term forces the encoded distributions to stay close to a standard Gaussian, regularizing both the mean (global structure) and the variance (local structure).
  • Without it, encoding as distributions alone is not enough: the model can ignore the distributions by shrinking variances toward zero or pushing means far apart, behaving like a plain autoencoder.
  • The result would be an irregular latent space lacking continuity and completeness: nearby latent points could decode to dissimilar or meaningless outputs, so sampling new points would not generate plausible data.
  • With the regularization, the latent space becomes regular: nearby points decode to similar content and overlapping clusters, enabling robust generation.

Word embeddings are central to modern NLP. Answer the following about how they are learned and what they capture.

Ch. 07 · Deep Learning · Text

a
Explain what a word embedding is and contrast it with a bag-of-words / TF-IDF representation, focusing on dimensionality, sparsity, and what distances between vectors mean.
Model answer
  • A word embedding is a dense, low-dimensional, real-valued vector assigned to each word, with the dimension typically much smaller than the vocabulary size.
  • Bag-of-words / TF-IDF produce one feature per vocabulary word, giving very high-dimensional and sparse vectors (mostly zeros), which are hard for ML algorithms and become even worse when used as sequences.
  • Embeddings instead are dense (few dimensions, mostly nonzero) and are learned directly from data.
  • In embedding space, distances (e.g. Euclidean or cosine) capture semantic similarity: related words sit close together, whereas in BoW/TF-IDF space the coordinates are just word-presence/weight counts and distances do not directly encode meaning.
  • Embeddings also support analogies via vector arithmetic, e.g. \( [\text{king}] - [\text{man}] + [\text{woman}] \approx [\text{queen}] \).
b
word2vec learns embeddings from a large corpus using a self-supervision task rather than hand-crafted vectors. Explain the core idea (context-based prediction) and describe what is kept from the network after training and why the hidden-layer size matters.
Model answer
  • Core idea: a word is defined by and correlated with its context, so we set up a prediction task between a focus word and its surrounding context words. This is self-supervision because the labels come for free from the text itself, no manual annotation is needed.
  • CBOW predicts the focus word from the surrounding context words; skip-gram predicts the context words from the focus word. Words are fed in one-hot encoded.
  • After training, the network itself is thrown away and only the hidden-layer weight matrix \( W \) is retained; each row of \( W \) is the embedding for one word (it reduces to a one-hot lookup into \( W \)).
  • The size of the hidden layer \( N \) equals the dimensionality of the resulting word vectors, so it directly controls how compressed/expressive the embeddings are. This is analogous to an auto-encoder performing dimensionality reduction, but with context as the learning signal.
  • The learned embeddings can then be transferred and reused as feature vectors for other downstream tasks.

Shallow node-embedding methods such as DeepWalk and node2vec were early attempts to bring deep learning to graphs, but graph neural networks (GNNs) were introduced to overcome their limitations.

Ch. 09 · Deep Learning · Graphs

a
Explain how DeepWalk produces node embeddings, and state precisely what node2vec adds on top of DeepWalk.
Model answer
  • DeepWalk is an unsupervised method that learns low-dimensional vector embeddings of nodes while preserving structural properties of the graph.
  • It generates many random walks on the graph; each walk is treated as a "sentence" and each visited node as a "word".
  • These walks are then fed to a skip-gram model (the same as word2vec): for a centre node it maximises the probability of the nodes appearing within a window around it, updating the embedding matrix \( \Phi \) by gradient descent.
  • node2vec keeps the same recipe (random walks + skip-gram) but replaces the uniform walk with a biased 2nd-order random walk controlled by two parameters \(p\) and \(q\).
  • The return parameter \(p\) governs the tendency to walk back to the previous node, and the in-out parameter \(q\) interpolates between BFS-like (structural) and DFS-like (community) exploration, so the walk (and resulting embeddings) can be steered.
b
Give two drawbacks of these shallow embedding approaches and explain how GNNs address them.
Model answer
  • Computational inefficiency / no parameter sharing: in DeepWalk/node2vec the encoder shares no parameters across nodes, so the number of parameters grows linearly with the number of nodes. GNNs share weights (e.g. a single learned message/update function applied at every node), keeping the parameter count independent of graph size.
  • Lack of generalisation (not inductive): shallow embeddings cannot handle dynamic graphs or infer embeddings for new, unseen nodes. GNNs (e.g. GraphSAGE) are inductive because their operations depend only on a node's neighbourhood, so they can produce embeddings for nodes not seen during training.
  • (Either of the following also counts) Limited representational ability: it is hard to incorporate node and edge features into shallow embeddings, whereas GNNs use both features and connectivity, combining feature matrices with message passing over the graph structure.
