1

Compare the three foundational white-box supervised techniques of this chapter: logistic regression, decision trees, and k-nearest neighbors.

Ch. 02 · Preprocessing

a
Characterize each technique on the parametric-to-data-driven spectrum, and describe the shape of the decision boundary each produces.
Model answer
  • Logistic regression is parametric: it learns a fixed set of weights and produces a single linear (diagonal) decision boundary with a smooth probability gradient.
  • Decision trees are data-driven: they recursively partition the feature space, giving a boundary made of axis-aligned rectangular regions.
  • k-NN is extremely data-driven (a lazy learner): the data itself is the model; the boundary is locally flexible, following the local arrangement of neighbors rather than a global functional form.
b
For each technique, state whether feature scaling is needed and whether it is sensitive or robust to outliers, and justify why.
Model answer
  • Logistic regression: needs scaling (and especially when regularized) and is sensitive to outliers, because it fits coefficients on the actual numeric magnitudes and extreme points pull the fit.
  • Decision trees: do not need scaling and are robust to outliers, because splits depend only on the ordering of values within a feature, so monotone rescaling and extreme magnitudes do not change the chosen thresholds.
  • k-NN: needs scaling and is sensitive to outliers/noise, because it is distance-based, so a feature on a larger scale or an extreme value dominates the neighbor computation.
2

AdaBoost combines weak learners over several rounds by reweighting both the data and the learners.

Ch. 04 · Trees & Ensembles

a
Describe one full round of AdaBoost: how the weighted error \(\varepsilon_{t}\) and the learner weight \(\alpha_{t}\) are computed, how the instance weights are then updated, and what stopping condition applies.
Model answer
  • Train a weak learner \(h_{t}\) on the current weighted data (labels \(y \in \{-1,+1\}\)).
  • Weighted error: \(\varepsilon_{t} = \sum_{i: y_{i} \neq h_{t}(x_{i})} D_{t}(i)\), the total weight on the misclassified instances.
  • Stopping condition: if \(\varepsilon_{t} \gt 0.5\) (worse than random), stop and discard that round.
  • Learner weight: \(\alpha_{t} = \tfrac{1}{2}\ln\frac{1-\varepsilon_{t}}{\varepsilon_{t}}\); smaller error gives larger \(\alpha_{t}\), i.e. a stronger vote.
  • Weight update: \(D_{t+1}(i) = D_{t}(i)\, e^{-\alpha_{t} h_{t}(x_{i}) y_{i}}\), then renormalise so the weights sum to 1. Misclassified instances (\(h_{t}(x_{i}) y_{i} = -1\)) get up-weighted; correctly classified ones get down-weighted.
  • Final prediction: \(H(x) = \mathrm{sign}\big(\sum_{t} \alpha_{t} h_{t}(x)\big)\).
b
Compute \(\alpha_{t}\) for a learner with weighted error \(\varepsilon_{t} = 0.25\), and explain in one sentence why a noisy/outlier-heavy dataset can be problematic for AdaBoost.
Model answer
  • \(\alpha_{t} = \tfrac{1}{2}\ln\frac{1-0.25}{0.25} = \tfrac{1}{2}\ln(3) \approx 0.549\).
  • Noisy points and outliers are repeatedly misclassified, so the reweighting keeps increasing their weight each round; the ensemble then over-focuses on these hard, possibly mislabeled instances, which hurts generalization.
3

Compare PCA, t-SNE, and UMAP as dimensionality-reduction techniques.

Ch. 06 · Unsupervised

a
Contrast PCA with t-SNE in terms of linearity, what structure they preserve, and whether new (unseen) points can be embedded.
Model answer
  • Linearity: PCA is a linear technique, computing principal components as linear combinations of the original features via the eigenvector decomposition of the covariance matrix (eigenvalue \( \lambda_j \) = explained variance of component j). t-SNE is non-linear, based on manifold learning.
  • Structure preserved: PCA preserves global variance directions and yields uncorrelated components, but cannot capture non-linear manifolds and can be hard to interpret. t-SNE preserves local similarities (small pairwise distances) by matching a Gaussian-based distribution \( p_{ij} \) in high-dim space to a Student-t distribution \( q_{ij} \) in the map, minimizing the KL divergence via gradient descent.
  • Embedding new points: PCA learns an explicit linear projection, so unseen points can be embedded. t-SNE learns a non-parametric mapping with no explicit input-to-map function, so unseen test points cannot be embedded directly, making it less suitable for predictive featurization (parametric t-SNE is an extension that addresses this).
b
Give three concrete advantages of UMAP over t-SNE.
Model answer
  • UMAP can embed new (unseen) points — its standard implementation provides a transform to project new data into the learned space — so it works well as general-purpose preprocessing. (The slides call this "parametric by default"; strictly, standard UMAP is non-parametric, while a separate Parametric UMAP variant learns an explicit neural mapping.)
  • UMAP scales better than t-SNE (and MDS) and performs better for mappings with dimensionality greater than 3, whereas most t-SNE implementations are limited to 2D or 3D.
  • UMAP can incorporate supervised labels in constructing the mapping, and is generally regarded as having nicer interpretation properties than t-SNE; it is recommended as a drop-in t-SNE replacement.

Spark and Hadoop MapReduce are both distributed processing approaches, but they differ fundamentally.

Ch. 08 · Big Data

a
Compare Hadoop MapReduce and Spark on (i) the execution model / handling of intermediate data and (ii) how each achieves fault tolerance. Explain why Spark is typically much faster.
Model answer
  • Execution model and intermediate data: MapReduce runs a rigid map -> shuffle -> reduce sequence and writes intermediate results to HDFS (disk) between steps. Spark replaces this with a DAG execution engine supporting cyclic data flow and in-memory computing, keeping intermediate RDDs in memory wherever possible.
  • Fault tolerance: MapReduce re-runs failed tasks, relying on the intermediate data persisted to HDFS. Spark instead tracks the lineage of transformations applied to each RDD; if a partition is lost on a node failure, the logged chain of transformations is simply recomputed.
  • Why Spark is faster: avoiding the constant write/read swapping to HDFS between stages and keeping data in memory (especially valuable for iterative workloads) makes Spark on the order of 10-100x faster than Hadoop MapReduce on the relevant benchmarks.
b
Explain why MapReduce was a poor fit for analytics/machine-learning workloads, and what abstraction Spark offers (beyond raw RDDs) that makes it more accessible to data scientists.
Model answer
  • MapReduce poor fit for analytics: most ML/analytics techniques are extremely hard to express as map-reduce; the constant in/out swapping to HDFS makes it slow, and it is poorly suited to iterative or 'online' tasks. The Mahout project's struggle to port algorithms illustrates this. Programming it is low-level (Java, 'most tutorials don't go further than word count').
  • Spark's higher-level abstraction: on top of RDDs, Spark SQL provides the DataFrame (and in Scala/Java, Dataset) API, an immutable distributed collection organised into named columns, manipulated through a high-level DSL or SQL and optimised by the Catalyst optimizer. Since Spark 2.0 this is the primary interface, hiding RDDs.
  • This is much closer to what data scientists already know (pandas-like data frames), and MLlib ('scikit-learn on Spark') provides scalable ML algorithms, featurization, pipelines and persistence on top, making predictive modelling far more accessible than hand-written map-reduce.
