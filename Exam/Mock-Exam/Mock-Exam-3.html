How does a linear autoencoder relate to PCA?

📖 Ch 5 · Deep Learning: Images & Ch 6 · Unsupervised

  1. Both find a best linear subspace, but the linear autoencoder imposes no orthogonality constraint and does not order components by variance.
  2. A linear autoencoder always produces orthogonal components ordered by decreasing variance, exactly like PCA.
  3. PCA can capture non-linear structure that a linear autoencoder cannot.
  4. The linear autoencoder has a unique solution, whereas PCA has infinitely many.

Which statement about agglomerative vs. divisive hierarchical clustering is correct?

📖 Ch 6 · Unsupervised

  1. Agglomerative clustering starts from the complete dataset and recursively splits it into partitions.
  2. Divisive clustering is bottom-up, merging singletons until one cluster remains.
  3. Divisive clustering is generally more computationally expensive because it must consider all possible ways to split a cluster.
  4. Both approaches require the number of clusters k to be fixed before building the hierarchy.

Which statement about the file formats discussed (AVRO, Parquet, Arrow) is correct?

📖 Ch 8 · Big Data

  1. Parquet is row-oriented and is mainly chosen for fast writing of complete records in ETL pipelines.
  2. Arrow is a binary on-disk file specification that competes with Parquet for long-term storage.
  3. AVRO is row-based, embeds its schema in the file, and supports schema evolution, making it well suited to ETL/integration.
  4. Parquet cannot evolve its schema, so a new column always requires rewriting every existing file from scratch.

The lecture distinguishes three aspects of "streaming" and argues about when online algorithms are actually needed. Which statement is correct?

📖 Ch 8 · Big Data

  1. Streaming data, streaming training, and streaming prediction are the same problem and are always solved together.
  2. In many cases one can ingest streaming data and predict online while training the model offline on historical data.
  3. The required re-training cadence is dictated mainly by how fast events arrive in the stream.
  4. Because volumes are huge, online (incremental) algorithms are essential for every streaming use case.
20

Which statement about feeding a graph's connectivity into a neural network is INCORRECT?

📖 Ch 9 · Deep Learning: Graphs

  1. An adjacency matrix is typically sparse, and many different adjacency matrices can encode the very same connectivity.
  2. A plain DNN fed an adjacency matrix is permutation invariant, so node ordering does not affect its output.
  3. Adjacency lists store each edge as a tuple \((i, j)\) and are a memory-efficient alternative to the adjacency matrix.
  4. Node, edge, and global features are comparatively easy to supply as feature matrices.
