Why does the move from multi-way to binary splits help decision-tree learning when using information gain?

📖 Ch 4 · Trees & Ensembles

  1. Binary splits guarantee pure leaves, so overfitting can no longer occur
  2. Information gain is biased toward splits that produce more subsets, and forcing two subsets removes this bias
  3. Binary splits make standardization of continuous features unnecessary
  4. Binary splits allow categorical features to be used without any encoding, unlike multi-way splits

7

Why is Mean Decrease in Impurity (MDI) feature importance considered untrustworthy in certain settings?

📖 Ch 3 · Evaluation

  1. It requires retraining the model once per feature, making it computationally infeasible.
  2. It is biased toward high-cardinality features and can rank a column of pure random numbers above a real feature when overfitting.
  3. It can only be applied to linear models, not tree ensembles.
  4. It is model-agnostic, so it ignores the structure that makes trees informative.

What is the main computational difference between XGBoost and a classical gradient boosting machine?

📖 Ch 4 · Trees & Ensembles

  1. XGBoost replaces decision trees with linear base learners
  2. XGBoost removes regularization to fit the training data more tightly
  3. XGBoost uses second-order loss information (gradient and Hessian) to optimise faster
  4. XGBoost trains all of its trees in parallel rather than sequentially
