Which statement about supervised versus unsupervised learning is INCORRECT?

📖 Ch 1 · Introduction

  1. Unsupervised learning does not require a target variable and aims to find structure in the data
  2. Clustering, anomaly detection and dimensionality reduction are all unsupervised techniques
  3. Supervised learning requires a target variable and relates predictors to that target
  4. Association rule mining is a supervised technique because it predicts which items a customer will buy
2

A team imputes missing values, fits scaling parameters, and builds Weights-of-Evidence groupings, then evaluates the model. Which practice is CORRECT?

📖 Ch 2 · Preprocessing

  1. The imputation median and scaling parameters should be recomputed on the test set so they match its distribution
  2. All these transformations are fit on the training data only and then re-applied identically to validation, test, and production data
  3. WoE/IV can be computed on the full dataset before splitting because they are just encodings
  4. Scaling parameters may be fit on the whole dataset, but imputation must be fit per split separately

Which statement about k-fold cross-validation is correct?

📖 Ch 3 · Evaluation

  1. Cross-validation replaces the need for a separate held-out test set.
  2. When comparing hyperparameter configurations, you should pick the one with the best single-fold score.
  3. It gives a pessimistically biased performance estimate because each fold trains on less than the full data.
  4. Preprocessing should be fitted once on all data before splitting into folds.
8

In a random forest, when is the random subset of candidate features selected?

📖 Ch 4 · Trees & Ensembles

  1. Once per tree, fixed for all of that tree's splits
  2. A fresh random subset is drawn at every individual split
  3. Once for the whole forest, shared by every tree
  4. Only when the bootstrap sample changes between trees
9

In AdaBoost a weak learner in a given round achieves a weighted error of \(\varepsilon_{t} = 0.5\). What happens?

📖 Ch 4 · Trees & Ensembles

  1. It receives the largest possible vote weight \(\alpha_{t}\)
  2. Its vote weight \(\alpha_{t} = \tfrac{1}{2}\ln\!\frac{1-\varepsilon_{t}}{\varepsilon_{t}}\) becomes \(0\), so it contributes nothing
  3. Its instance weights are reset to uniform and the round restarts
  4. Its vote weight becomes negative, flipping its predictions

Out-of-bag (OOB) validation in bagging works because, on average, each bootstrap sample leaves out roughly which fraction of the training instances?

📖 Ch 4 · Trees & Ensembles

  1. About \(50\%\)
  2. About \(37\%\)
  3. About \(10\%\)
  4. About \(0\%\), since a \(100\%\) bootstrap selects every instance
11

Why do sigmoid and tanh activations tend to cause vanishing gradients in deep networks?

📖 Ch 5 · Deep Learning: Images

  1. Their outputs are unbounded, so activations grow without limit toward the input layers.
  2. Their derivatives are at most \(1\) and often much smaller, so repeated multiplication shrinks gradients toward the early layers.
  3. They are not differentiable, so backpropagation cannot compute gradients through them.
  4. They set roughly 40% of neurons permanently to zero activation.
12

Which statement about dropout is correct?

📖 Ch 5 · Deep Learning: Images

  1. At test time, all neurons are kept but the network is run multiple times and the outputs averaged.
  2. During training, the same fixed subset of neurons is dropped for every example.
  3. Validation loss can appear better than training loss because validation is measured without dropout.
  4. Dropped neurons still pass their gradients backward to keep all weights updating.
13

Which statement about generative adversarial networks (GANs) is correct?

📖 Ch 5 · Deep Learning: Images

  1. Mode collapse occurs when the generator produces a very narrow set of nearly identical outputs.
  2. GAN training is guaranteed to converge because both networks minimize the same loss.
  3. The discriminator generates fake samples from a noise vector.
  4. A perfect discriminator accelerates the generator's learning by providing strong gradients.
14

Which statement correctly distinguishes k-means from EM-based (Gaussian-mixture) clustering?

📖 Ch 6 · Unsupervised

  1. k-means makes hard assignments while EM makes soft, probabilistic assignments of instances to clusters.
  2. EM does not require the number of clusters k to be specified, unlike k-means.
  3. Unlike k-means, EM has no problem with local optima.
  4. EM, unlike k-means, naturally handles non-convex cluster shapes.
15

Which statement about t-SNE is correct?

📖 Ch 6 · Unsupervised

  1. t-SNE is itself a clustering algorithm that outputs cluster labels.
  2. In a t-SNE map, the relative size of clusters and the distance between clusters are reliably meaningful.
  3. It learns a non-parametric mapping, so unseen test points cannot be directly embedded into an existing map.
  4. Its low-dimensional similarities are computed with a Gaussian kernel, the same as in the high-dimensional space.

17

What is a key difference between an LSTM cell and a GRU cell?

📖 Ch 7 · Deep Learning: Text

  1. An LSTM passes two values to the next step (a cell state and a hidden state), whereas a GRU passes only one
  2. A GRU has no gates, relying purely on a tanh recurrence
  3. An LSTM trains faster and works better than a GRU on small datasets
  4. Only the GRU can model long-range dependencies; the LSTM cannot
18

Map over genre yields drama \(\to [100, 200]\). A naive reduce computes \(\frac{100+200}{2}=150\). Later a straggling mapper emits drama \(\to 300\) and the partial result \(150\) is re-reduced with \(300\). What value does the naive reduce now produce, and is it correct?

📖 Ch 8 · Big Data

  1. It produces \(200\), which is the correct mean of \(100, 200, 300\).
  2. It produces \(225\), which is wrong because the partial average of two records is weighted as if it were one.
  3. It produces \(150\), because reduce ignores any values that arrive after the first reduction.
  4. It produces \(450\), the running sum, which is correct because averaging happens only at the very end.
19

Which statement about Spark Streaming (DStreams) versus Spark Structured Streaming is correct?

📖 Ch 8 · Big Data

  1. A DStream is internally represented as a single continuously-growing RDD that is never split into intervals.
  2. Structured Streaming materialises the entire unbounded table in memory before producing any output.
  3. Structured Streaming treats the live stream as an unbounded table and lets you write batch-like queries on it.
  4. Spark Streaming is built directly on the Spark SQL engine, whereas Structured Streaming is built on raw RDDs.
20

Which statement correctly distinguishes plain PageRank from Personalized PageRank as used for network-based inference?

📖 Ch 9 · Deep Learning: Graphs

  1. Plain PageRank teleports the random walker to a node chosen uniformly at random, whereas Personalized PageRank teleports according to a non-uniform distribution biased toward a class of interest.
  2. Plain PageRank uses label information of the target class, whereas Personalized PageRank ignores labels entirely.
  3. Both variants teleport uniformly; they differ only in the embedding dimension they produce.
  4. Personalized PageRank forbids the walker from ever resting at a vertex, unlike plain PageRank.
