1

Core difference between bagging and boosting? (Bagging vs. Boosting)

Answer B
  1. Both train base learners sequentially, each correcting the previous one's errors.FALSE
  2. Bagging trains learners independently (in parallel) and mainly reduces variance; boosting trains them sequentially and mainly reduces bias.TRUE
  3. Bagging mainly reduces bias, while boosting mainly reduces variance.FALSE
  4. Bagging reweights misclassified instances, while boosting uses bootstrap samples.FALSE
Why Bagging (e.g. random forest) trains many independent models on bootstrap samples and averages them, cancelling their idiosyncratic errors → it cuts variance while leaving bias roughly unchanged. Boosting (AdaBoost, GBM) trains learners sequentially, each fixing the ensemble’s current mistakes, which mainly drives down bias. (A describes boosting only; C swaps the two effects; D swaps the two mechanisms.)
2

Distinguish AdaBoost from gradient boosting. (Bagging vs. Boosting)

Answer B
  1. AdaBoost fits each new learner to the residuals; gradient boosting reweights misclassified instances.FALSE
  2. AdaBoost increases the weights of misclassified instances; gradient boosting fits each new learner to the negative gradient (residuals) of the loss.TRUE
  3. Both are bagging methods that average independently trained deep trees.FALSE
  4. Both require fully grown (unpruned) trees as base learners.FALSE
Why Both are boosting methods, but they “focus on mistakes” differently. AdaBoost re-weights: misclassified instances get larger weights so the next learner concentrates on them. Gradient boosting instead fits each new tree to the negative gradient of the loss (the residuals). Option A states these exactly the wrong way round; C and D are simply false (they are sequential, and use weak/shallow learners).
3

Which hyperparameter sets t-SNE’s local neighborhood size? (t-SNE vs. UMAP)

Answer B · Perplexity
  1. Learning rateFALSE
  2. PerplexityTRUE
  3. min_distFALSE
  4. Number of estimatorsFALSE
Why Perplexity is t-SNE’s key knob — loosely the effective number of neighbors, set by the variance of each point’s conditional Gaussian. Learning rate is an optimisation setting, min_dist is a UMAP parameter, and “number of estimators” belongs to tree ensembles.
4

Which statement about t-SNE and UMAP is correct? (t-SNE vs. UMAP)

Answer B
  1. Both are linear projection methods, like PCA.FALSE
  2. UMAP typically preserves global structure better and scales to large datasets faster than t-SNE.TRUE
  3. t-SNE can project new, unseen points into an existing embedding, whereas UMAP cannot.FALSE
  4. UMAP has no hyperparameters, while t-SNE has perplexity.FALSE
Why UMAP is generally faster, scales better, tends to preserve global structure better, and — unlike t-SNE — can transform new out-of-sample points via a learned mapping. Both are nonlinear (so A is false); C reverses the out-of-sample property (it’s UMAP that can embed new points); and D is false because UMAP has hyperparameters n_neighbors and min_dist.
5

Which statement about Lasso (L1) and Ridge (L2) is correct? (Lasso vs. Ridge)

Answer C
  1. Ridge drives some coefficients exactly to zero, performing automatic feature selection.FALSE
  2. Lasso penalizes the sum of squared coefficients, while Ridge penalizes the sum of absolute values.FALSE
  3. Lasso can set some coefficients exactly to zero (feature selection), whereas Ridge only shrinks coefficients toward zero without eliminating them.TRUE
  4. Both methods always keep every feature with a non-zero coefficient.FALSE
Why The L1 penalty’s diamond-shaped constraint has corners on the axes, so Lasso drives some coefficients exactly to zero → built-in feature selection. Ridge’s circular L2 constraint only shrinks coefficients smoothly toward (never exactly to) zero. A and B swap the two methods; D is false for Lasso.
6

Many correlated predictors; want a stable model keeping all of them — which approach? (Lasso vs. Ridge)

Answer B · Ridge
  1. Lasso (L1), because it keeps exactly one feature from each correlated group and zeros the rest.FALSE
  2. Ridge (L2), because it shrinks coefficients smoothly and spreads weight across correlated predictors, giving stable estimates.TRUE
  3. Ordinary least squares with no penalty, because regularization cannot handle correlation.FALSE
  4. Lasso (L1), because it assigns infinite weight to correlated features.FALSE
Why With highly correlated predictors that you want to keep, Ridge is preferred: it spreads weight across the correlated group and produces stable estimates. Option A is actually a true description of lasso’s tendency to pick one and drop the rest — but that contradicts the goal of retaining all features, so it is wrong for this question. C and D are false. (Elastic Net is the natural compromise when you want both selection and grouping.)

Topic primers

reference

Deeper background on each examined topic.

Topic 1

Bagging vs. Boosting

Both are ensemble methods that combine many weak/unstable base learners (usually trees), but they attack different parts of the error.

BaggingBoosting
HowTrain models independently on bootstrap samples; combine by averaging / majority vote.Train models sequentially, each correcting the ensemble’s current errors; combine by weighted sum.
TrainingParallel (independent).Sequential (each depends on the last).
Mainly reducesVariance — averages deep, low-bias / high-variance trees.Bias — boosts shallow, high-bias weak learners.
“Focus on errors”—AdaBoost: re-weight misclassified points. Gradient boosting: fit residuals / negative gradients with a learning rate.
ExamplesRandom Forest (bagging + random feature subsets).AdaBoost, GBM, XGBoost, LightGBM.
OverfittingRobust — more trees just plateau.Can overfit with too many rounds / noisy data; control via learning rate, early stopping, depth, regularization.
💡Memory hook: Bagging = parallel + independent + averaging → cuts variance. Boosting = sequential + error-correcting → cuts bias (at higher overfitting risk).
Topic 2

t-SNE vs. UMAP

Both are nonlinear neighbor-embedding techniques for visualising high-dimensional data in 2-D/3-D: model neighbor similarities in high-D, then find a low-D layout that preserves them. Both are stochastic.

t-SNEUMAP
ObjectiveMinimise KL divergence between high-D and low-D pairwise affinities (Gaussian → heavy-tailed Student-t).Cross-entropy on a fuzzy simplicial set (weighted neighbor graph) from manifold theory.
Key knob(s)Perplexity (≈ effective # of neighbors).n_neighbors (local↔global) & min_dist (cluster tightness).
StructureGreat local structure; inter-cluster distances not meaningful.Preserves local and global structure better.
Speed / scaleSlower; non-parametric — can’t embed new points (must re-run).Faster, scales better; can transform new points into an existing embedding.
💡Use t-SNE for clean local cluster separation in a one-off picture; use UMAP for speed, scalability, more trustworthy global geometry, and projecting new data.
Topic 3

Lasso vs. Ridge

Both fight overfitting by adding a penalty on coefficient size to the loss — the difference is how they penalise, which changes their behaviour completely.

Ridge (L2): min RSS + λ Σ βj²      Lasso (L1): min RSS + λ Σ |βj|
Lasso (L1)Ridge (L2)
PenaltySum of absolute values.Sum of squared values.
Effect on coeffsDrives some exactly to zero → feature selection, sparse model.Shrinks all toward zero but keeps them all.
GeometryDiamond constraint — corners on the axes → hits zeros.Circle constraint — smooth, no corners → no exact zeros.
Correlated featuresArbitrarily picks one, zeros the rest.Spreads weight across them → stable under multicollinearity.
Use whenMany irrelevant features; want selection / interpretability.Most features useful; strong multicollinearity.

Elastic Net mixes both penalties — lasso-style selection plus ridge-style stability and a grouping effect for correlated predictors. For both, λ is tuned by cross-validation, features should be standardised first, and λ = 0 recovers OLS.

Confusion matrix, precision, recall & the ROC curve at a threshold

Definitions

A classifier outputs a score per instance; a threshold t turns scores into labels: predict positive if score ≥ t. Cross-tabulating predictions against truth gives four counts — TP, FP, TN, FN:

Precision = TP / (TP + FP)   — of predicted positives, how many are right Recall = TPR = TP / (TP + FN) — of real positives, how many were caught FPR = FP / (FP + TN) = 1 − specificity

Worked example — 8 scored instances

Four actual positives (A, B, D, F) and four negatives (C, E, G, H), sorted by descending score:

Precision = 3/(3+1) = 0.75    Recall = 3/(3+1) = 0.75    FPR = 1/(1+3) = 0.25

Building the ROC curve

The ROC curve plots TPR (y) against FPR (x) as the threshold sweeps from high (nothing positive) to low (everything positive). Each threshold just below a distinct score gives one point; with 4 positives and 4 negatives, each correct catch is a +0.25 step up and each false alarm a +0.25 step right:

Reading it: the dashed diagonal is a random classifier (TPR = FPR); a good model bows toward the top-left corner. AUC = 0.5 (random) to 1.0 (perfect) and equals the probability a random positive outranks a random negative. Here AUC = 0.8125 — a good ranking model. ROC is threshold-free and, because it uses only within-class rates, robust to class imbalance in a way precision is not.

Key points
  • Threshold turns scores into labels → tally TP/FP/TN/FN into the confusion matrix.
  • Precision = TP/(TP+FP); Recall = TPR = TP/(TP+FN); FPR = FP/(FP+TN).
  • ROC = TPR vs FPR while sweeping the threshold across every distinct score.
  • Diagonal = random; bow to top-left = good; AUC summarises ranking quality.

Permutation feature importance — algorithm, drawbacks, and why it couldn’t be applied

The algorithm

Permutation importance measures how much a model relies on a feature by destroying that feature’s link to the target and watching performance drop:

  • Baseline. With the already-fitted model and labelled data, compute a baseline score s (AUC, accuracy, R²…).
  • Permute one feature. Randomly shuffle column j — this keeps its marginal distribution but breaks its association with the target and other features.
  • Re-score. Run the same fitted model (no retraining) on the corrupted data → score sj.
  • Importance = drop: s − sj. Big drop → heavily used feature.
  • Repeat & average over several shuffles for a stable estimate.

It is model-agnostic and post-hoc — it only needs to call predict().

⛔Why it couldn’t be applied on this exam: permutation importance is a property of a fitted model, not of data. Every step needs predictions from the trained function, and importance is defined as a change in a performance score (needs predictions + true labels). With only a data matrix and no trained algorithm provided, there is no baseline s and no permuted score sj to compare — so there is simply nothing to compute.

Drawbacks

  • Needs a fitted model + labelled scoring data — can’t be computed from data alone (the reason it failed here).
  • Correlated features dilute importance: shuffling one is “covered” by its correlated partner, so the drop is small and importance is understated/split across the group.
  • Unrealistic rows: shuffling one column independently creates out-of-distribution combinations the model never saw, which can mislead.
  • Metric-dependent and computationally costly (≈ #features × #repeats re-scorings).
  • Train vs test: on the training set it rewards overfit features; use a held-out test set for generalisation-relevant importance.

What makes a random forest “random” — and random forest vs. isolation forest

The two sources of randomness

A random forest injects randomness in two places, and that double randomisation is the point: it decorrelates the trees so averaging them actually reduces variance.

  • Bagging of rows: each tree is trained on a bootstrap sample (random rows with replacement).
  • Random feature subspace at each split: only a random subset of features (max_features, e.g. √p) is considered when choosing each split — so no single dominant feature makes all trees alike.

Within those constraints each split is still chosen greedily on an impurity criterion (Gini / entropy / variance), and predictions combine by majority vote or averaging.

Random forest vs. isolation forest

They share the “many random trees” architecture but are built for opposite goals:

Random forestIsolation forest
ParadigmSupervisedUnsupervised
GoalPredict a label (classification / regression)Detect anomalies / outliers
Split choiceBest split on a label-based impurity criterion, over a random feature subsetFully random: random feature + random split value — no label, no criterion
TreesDeep trees that fit the targetShallow random partitions grown just to isolate points
OutputVote / average of leaf predictionsAnomaly score from average path length to isolate a point
IntuitionDecorrelated trees averaged to cut varianceAnomalies are “few and different,” so they isolate in fewer splits → shorter path → higher score
💡Key contrast: in a random forest the randomness is a regularizer serving accurate supervised prediction; in an isolation forest the randomness is the detector itself — outliers are easy to isolate, so short average path lengths flag them, with no labels involved.
