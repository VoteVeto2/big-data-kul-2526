Ch 1 · Analytics & Big Data Foundations

framing

Must remember

  • 3 components of data science: Data · Algorithm · Purpose.
  • 4 quality criteria: Valid=generalizable, Useful=actionable, Unexpected=interesting, Understandable=interpretable.
  • ML = generalizable correlations, NOT causation.
  • "Binary classification forms the majority of applicative tabular settings."
  • Most business ML is really about automation, not ML. (Google Rule #1: don't fear launching without ML.)
  • You don't need big data to do analytics, and big data ≠ doing analytics (orthogonal).
  • Non-parametric ≠ no parameters — it means no fixed functional form (trees, ensembles).

Ch 2 · Preprocessing & Basic Techniques

white-box

Preprocessing essentials

Golden rules

  • Everything over multiple instances is part of the pipeline — fit on train only, re-apply to test/prod. Never impute a new median.
  • Remove future variables (leakage); set aside hold-out ASAP.
  • Median (numeric) + mode (categorical) imputation; add a missing indicator.
  • Truncate valid outliers at \(\mu\pm 3\sigma\); treat invalid outliers as missing.
  • WoE/IV use the target → after the train/test split!
  • Trees are robust to outliers & scaling; standardization is useless for trees.

Transforms

Standardization\[ x_{new}=\frac{x-\mu}{\sigma} \]
Normalization (min-max)\[ x_{new}=\frac{x-x_{min}}{x_{max}-x_{min}} \]
Weight of Evidence\[ \text{WoE}_{cat}=\ln\!\frac{p_{c1,cat}}{p_{c2,cat}} \]
Information Value (screen if \(\gt 0.1\))\[ \text{IV}=\sum (p_G-p_B)\cdot\text{WoE} \]
Categorical encoding (k=3 levels)#featuresNote
Integer (high=1,med=2,low=3)1imposes false order & spacing
One-hot (dummy)3 (or 2, drop one)regression; drop one to avoid multicollinearity
Binary2 (\(\approx\log_2 k\))compact, less common

Linear & logistic regression

OLS objective (SSE, convex)\[ \arg\min_{\beta}\sum_{i=1}^{n}(y_i-\hat{y}_i)^2 \]
Sigmoid / logistic\[ f(z)=\frac{1}{1+e^{-z}} \]
Logit (log-odds, linear)\[ \ln\!\frac{P(\text{yes})}{P(\text{no})}=\beta_0+\sum_i\beta_i x_i \]
Odds-ratio: \(X_i{+}1\)\[ \text{odds}\mid_{X_i+1}=\text{odds}\mid_{X_i}\cdot e^{\beta_i} \]
Binary cross-entropy (log loss)\[ -\big(y\log p+(1-y)\log(1-p)\big) \]

LogReg is a classification technique (binary target), predictions in \(]0,1[\). Linear boundary, well-calibrated, sensitive to outliers & curse of dimensionality. \(e^{\beta_i}\gt 1\Rightarrow\) odds increase.

Regularization

Lasso (L1) — sparsity, selection\[ \sum(y_i-\hat{y}_i)^2+\lambda\sum_{j=1}^{p}|\beta_j| \]
Ridge (L2) — shrink, grouping\[ \sum(y_i-\hat{y}_i)^2+\lambda\sum_{j=1}^{p}\beta_j^2 \]
Elastic net\[ \sum(y_i-\hat{y}_i)^2+\lambda_1\sum|\beta_j|+\lambda_2\sum\beta_j^2 \]

L1 vs L2

  • Lasso: diamond constraint, hits corners → coefficients exactly 0 (selection "for free"); fails when \(n\lt p\); no grouping.
  • Ridge: circular constraint, shrinks but rarely to 0; handles correlated vars (grouping).
  • No penalty on intercept. Must normalize first. Pick \(\lambda\) by cross-validation.
  • Stepwise selection (Harrell): biased \(R^2\), bad p-values → use proper train/valid/test.

Decision trees & impurity

Entropy (0 = pure)\[ H(S)=-\sum_{x\in X}p(x)\log_2 p(x) \]
Gini (default in most libs)\[ \text{Gini}(S)=1-\sum_{x\in X}p(x)^2 \]
Classification error\[ \text{Err}(S)=1-\max_{x\in X}p(x) \]
Information Gain (split quality)\[ \text{IG}(A,S)=H(S)-\sum_{t\in T}\frac{|t|}{|S|}H(t) \]
  • Entropy peaks at 1.0, Gini & ClassErr at 0.5 (all at \(p=0.5\)).
  • ID3: categorical only, greedy, overfits. C4.5: continuous + missing + pruning. CART: regression trees.
  • Binary splits avoid IG bias toward many subsets; allow reusing attributes.
  • Trees: not well-calibrated, unstable; axis-aligned rectangles; struggle with linear relations.
Regression tree & k-NN
Regression tree: leaf = mean of label, splits minimize SSE (within-variance) → piecewise-constant. k-NN = non-parametric, lazy ("the data is the model"), distance-based → normalize; classify by majority vote, regress by inverse-distance-weighted average. Parametric → data-driven → extremely data-driven: LogReg → trees → k-NN.

Ch 3 · Evaluation & Interpretability

generalization
It's all about generalization
Split first (before preprocessing), stratified, prefer out-of-time. 100% train accuracy is trivial (full tree, k=1 NN, DB). FP = Type I, FN = Type II. Goal of test set: catch failure cases.

Confusion-matrix metrics (all threshold-dependent)

Accuracy\[ \frac{TP+TN}{\text{total}} \]
Recall = Sensitivity = TPR\[ \frac{TP}{TP+FN} \]
Precision = PPV\[ \frac{TP}{TP+FP} \]
Specificity = TNR\[ \frac{TN}{TN+FP} \]
F1 (harmonic mean)\[ \frac{2\,P\cdot R}{P+R} \]
Balanced accuracy\[ \tfrac{1}{2}(\text{recall}+\text{specificity}) \]

Cost-sensitive learning

Expected loss of class \(j\)\[ l(x,j)=\sum_k p(k\mid x)\,C(j,k) \]

Classify positive iff \(l(x,1)\lt l(x,0)\).

Cost-sensitive threshold\[ T_{CS}=\frac{C(1,0)-C(0,0)}{\big(C(1,0)-C(0,0)\big)+\big(C(0,1)-C(1,1)\big)} \]

Symmetric unit costs → \(T_{CS}=0.5\). Sampling to balance ⇔ inverse-class-imbalance cost matrix.

Sampling traps
Resample ONLY the training set. After resampling, test accuracy is usually lower than always-majority (you trade FPs for TPs). SMOTE = synthetic points between a minority instance and its k-NN; can't invent future positives; often NOT worth it for tree ensembles / DL.

Threshold-independent metrics

  • ROC: TPR (y) vs FPR (x). AUC = P(random positive scores higher than random negative); coin flip = 0.5.
  • PR curve: precision vs recall — better under imbalance (TNs inflate ROC, precision ignores TN).
  • AR = \(2\cdot\text{AUC}-1\) (Gini); CAP = Lorenz = power curve. Lift of 1 = random.
  • h-measure (Hand 2009): coherent AUC alternative (AUC uses different cost dists per classifier).
KS separation\[ \text{KS}=\max_s\big|P(s\mid y{=}1)-P(s\mid y{=}0)\big| \]
Mahalanobis\[ M=\frac{|\mu_p-\mu_n|}{\sigma} \]
Divergence\[ D=\frac{(\mu_p-\mu_n)^2}{\tfrac{1}{2}(\sigma_p^2+\sigma_n^2)} \]

Regression metrics & cross-validation

\(R^2\)\[ R^2=1-\frac{\text{SSE}}{S_{yy}} \]
Adjusted \(R^2\) (can decrease)\[ R^2_a=1-(1-R^2)\frac{n-1}{n-k} \]
RMSE\[ \text{RMSE}=\sqrt{\tfrac{1}{n}\textstyle\sum(y_i-\hat{y}_i)^2} \]
Cross-validation
CV is for tuning; the test set is for true evaluation. Compare configs by the average over folds (not best fold). Preprocessing fit per fold. CV is pessimistically biased (smaller train sets); LOOCV ≈ unbiased. Bias–variance: complexity↑ → bias²↓, variance↑. One-vs-all = k classifiers; one-vs-one = \(k(k-1)/2\).

Random Forest classic traps
TWO randomness sources: bootstrap rows AND random feature subset at EVERY split (not once per tree). Heuristic \(M=\lfloor\sqrt{N}\rfloor\) classification, \(\lfloor N/3\rfloor\) regression. Bootstrap 100% leaves out ~37% (\(e^{-1}\)) → OOB. No pruning needed. OOB ≠ replacement for a test set.

AdaBoost (labels \(\in\{-1,+1\}\))

Learner weight\[ \alpha_t=\tfrac{1}{2}\ln\frac{1-\varepsilon_t}{\varepsilon_t} \]
Reweight\[ D_{t+1}(i)=D_t(i)\,e^{-\alpha_t h_t(x_i)y_i} \]
Output\[ H(x)=\text{sign}\Big(\sum_{t=1}^{T}\alpha_t h_t(x)\Big) \]

Stop if \(\varepsilon_t\gt 0.5\). Minimizes exponential loss.

Gradient Boosting

MSE gradient = residual\[ -\frac{\partial L_{MSE}}{\partial F}=\tfrac{2}{n}\,(y-F(x)) \]
  • Fits learner to negative gradient of loss; residual-fitting is the MSE special case.
  • XGBoost = 2nd-order (gradient + Hessian) + L1/L2; objective = loss + complexity.
  • Tree growth: XGBoost level-wise, LightGBM leaf-wise, CatBoost symmetric.
  • CatBoost: native categoricals + ordered boosting (less leakage).
Deep learning for tabular
"In most cases: no(t really)" — GBMs usually win honest benchmarks (no spatial/sequential structure, limited data). MLPs underperform GBMs. Tabular nets (TabNet, FT-Transformer, SAINT) treat features as tokens with self-attention. TabPFN = Transformer pretrained on synthetic tasks, in-context Bayesian inference, no tuning; v2/v2.5 rival GBMs.

Ch 5 · Deep Learning I — Images (CNNs)

unstructured

Foundations

Sigmoid perceptron\[ o(X)=\frac{1}{1+e^{-(\sum_i w_i x_i+w_b)}} \]
GD update\[ w_{i,t+1}=w_{i,t}-\lambda\frac{\partial L}{\partial w_i} \]
Sigmoid derivative (→ vanishing)\[ \sigma'(o)=\sigma(o)\big(1-\sigma(o)\big) \]

Single perceptron can't solve XOR; MLP can. 1 epoch = full pass. Bias = intercept.

Activations & init

ReLU (hidden only)\[ f(x)=\max(0,x) \]
Softmax (multiclass)\[ \sigma(z)_i=\frac{e^{z_i}}{\sum_j e^{z_j}} \]
  • Sigmoid/tanh → vanishing gradients (deriv \(\lt 1\)). ReLU can "die" (~40% if LR high).
  • He (\(2/\text{fan\_in}\)) for ReLU; Glorot (\(1/\text{fan\_avg}\)) tanh/softmax; LeCun (\(1/\text{fan\_in}\)) SELU.
  • Output: sigmoid/softmax. Multilabel = per-output sigmoid.
CNN output size (memorize!)\[ \text{out}=\frac{W-F+2P}{S}+1 \]

W=input, F=filter, P=padding, S=stride. e.g. \(W{=}224,F{=}11,S{=}4,P{=}0\Rightarrow(224-11)/4+1=55\).

CNN facts

  • Weight sharing (same kernel all positions) = key efficiency. Kernel depth = #input channels; #filters = output depth.
  • Pooling: same depth as before; global pooling → single value (alt to dense).
  • VGG ≈ 140M params; SqueezeNet ≈ 50× fewer than AlexNet; GoogLeNet = Inception.
  • ResNet skip: \(F(x)+x\) fights vanishing gradients; no FC at end.
  • ViT: pure transformer on patches; CNNs not necessary. CNNs biased to texture.

Generative & regularization

Focal loss\[ \text{FL}(p_t)=-(1-p_t)^{\gamma}\log(p_t) \]
VAE loss\[ \lVert x-\hat{x}\rVert^2+\text{KL}\big[N(\mu_x,\sigma_x),N(0,I)\big] \]
  • Dropout: zero nodes (train); scale at test; val loss can look better (no dropout).
  • Linear autoencoder ≈ PCA but no orthogonality / variance ordering.
  • GAN: G vs D (0=fake); hard (mode collapse, no convergence). WGAN = Earth Mover's.
  • Diffusion: denoise Gaussian noise; U-Net. Stable Diffusion = latent diffusion.

t-SNE traps
NOT a clustering technique. Step 1 = Gaussian, step 2 = Student-t (fat tails), no perplexity in step 2. Cluster size & inter-cluster distance are meaningless; random data can look structured. PCA: eigendecomp of covariance, uncorrelated PCs, \(\lambda_j\) = explained variance.

Anomaly detection & association rules

Support\[ \frac{|\{t\in T:X\subseteq t\}|}{|T|} \]
Confidence \(=P(Y\mid X)\)\[ \frac{\text{Supp}(X\cup Y)}{\text{Supp}(X)} \]
Lift (=1 indep, >1 dep)\[ \frac{\text{Supp}(X\cup Y)}{\text{Supp}(X)\,\text{Supp}(Y)} \]
Apriori & anomalies
Apriori: (1) min support → frequent itemsets, (2) min confidence → rules. Downward closure: subsets of frequent are frequent → prune. Power set \(=2^{|I|}-1\). Isolation forest: random splits, outliers = short path length. LOF: k-NN density. (V)AE: high reconstruction error → anomaly.

Ch 7 · Deep Learning II — Text & Transformers

NLP

Classical featurization

TF (÷ document length!)\[ \text{TF}(t,d)=\frac{f(t,d)}{|d|} \]
IDF (note the 1+)\[ \text{IDF}(t)=1+\log\frac{|c|}{|\{d:t\in d\}|} \]
TF-IDF\[ \text{TFIDF}(t,d)=\text{TF}(t,d)\cdot\text{IDF}(t) \]

BM25 saturates TF (nonlinear), better length norm; can beat embeddings.

word2vec & embeddings

Analogy magic\[ [\text{king}]-[\text{man}]+[\text{woman}]\approx[\text{queen}] \]
  • CBOW: context→word, averaging, less data. Skip-gram: word→context, pass-through, better quality, more data.
  • Keep hidden weights W after training (size = vector dim).
  • GloVe = global co-occurrence. fastText = subword n-grams → OOV/misspelled words.

RNN family

RNN cell\[ a^{\langle t\rangle}=\tanh(W_{ax}x^{\langle t\rangle}+W_{aa}a^{\langle t-1\rangle}+b_a) \]
  • Training = BPTT (weight sharing across time). In practice few steps back (vanishing).
  • LSTM (1997): passes TWO values (cell \(C_t\) + hidden \(h_t\)). GRU (2014): one; faster, less data.
  • Bidirectional ≠ cheating with the future.

Attention & transformers

Scaled dot-product (Vaswani 2017)\[ Z=\text{softmax}\!\Big(\frac{QK^{\top}}{\sqrt{d_k}}\Big)V \]
  • Self-attention: Q,K,V from same set. Attention mixes between tokens; FFN processes each independently.
  • GPT = decoder-only, masked self-attention (why it generates).
  • Positional encoding added to embeddings (no inherent order). Multi-head = multiple Q/K/V sets, concat + \(W^O\).
  • Transformer gen cost \(\approx L^2\); RNN = linear. RLHF: human feedback as loss (BLEU/ROUGE).

Hadoop

  • 4 modules: Common, HDFS (storage), MapReduce (compute), YARN (resources).
  • HDFS: name node + data nodes; block = 64MB; replicates for fault tolerance. Storage, not computing!
  • Map = list→list; Reduce = list→scalar.

  • RDD = resilient distributed dataset; transformations (lazy, new RDD) vs actions; fault tolerance via lineage.
  • Runs on YARN/Mesos/standalone; many storages (HDFS, S3, Cassandra).
  • SparkSQL/DataFrame = schema on RDDs; Catalyst optimizer. .toPandas() pulls to client.
  • MLlib, GraphX, Spark Streaming. Streaming: Kafka, Flink.
MapReduce reduce-associativity trap
Reduce must give correct results when called multiple times on partial results & emit the same structure as map. Naive averaging is WRONG: store value as (running average, count) and compute a weighted mean so re-reducing partial results stays correct.

Communities & embeddings

  • Girvan–Newman: divisive, remove highest-betweenness edge. Louvain: greedy modularity + supernodes.
  • k-core = each vertex ≥ k connections in subset; maximal clique = not in a larger clique.
  • DeepWalk: random walk + skip-gram (unsupervised). node2vec: biased 2nd-order walk (p, q).
  • Shallow embeddings: no param sharing, grow linearly with #nodes, not inductive.
  • Homophily / Markov property = outcome from 1-hop neighbors ("guilt by association").

GNN message passing

Generic message passing\[ x_i^{k}=\gamma^{k}\Big(x_i^{k-1},\;\underset{j\in N(i)}{\text{AGG}}\;\phi^{k}(x_i^{k-1},x_j^{k-1},e_{j,i})\Big) \]
  • φ = message, AGG = aggregate, γ = update. ≡ convolution on irregular domains; CNN = special case (grid).
  • Stacking layers widens receptive field. Spectral = special case (no neighbor agg).
  • GraphSAGE = inductive, sample+aggregate, CONCAT + L2-norm. GAT = attention.
Graph validation = leakage central trap
Whole-graph features violate "engineer after split". Transductive (same graph, masked nodes) leaks — masked nodes still message-pass. Inductive (separate graph) better, but cutting edges removes info. Personalized PageRank uses labels → leaks more than label-agnostic centralities.

Graph DBs (Cypher)

  • Graph DB = NoSQL but goes toward MORE relational modeling (hyper-relational; traversal/pattern matching).
  • Nodes (), type (b:Book), edges --/-->, type in [:WROTE].
  • Transitive paths via *min..max — hard in SQL.
  • Neo4j optimized for OLTP; GDS library adds analytics.

ACID vs BASE

  • ACID (relational): Atomicity, Consistency, Isolation, Durability.
  • BASE (NoSQL): Basically Available, Soft state, Eventual consistency.
  • Eventual consistency = replicas converge after a transaction, not continuously.
  • NewSQL = relational + horizontal scaling + JSON (CockroachDB).

Classic Exam Traps

don't slip
  • Bagging reduces variance, boosting reduces bias — never swap. Bagging = parallel/diversity; boosting = sequential/cooperation, no bootstrap.
  • RF feature subset sampled at EVERY split, not once per tree. Bootstrap 100% still leaves ~37% out (OOB).
  • Non-parametric ≠ no parameters (no fixed functional form). Linear reg = parametric; trees/k-NN = non-parametric.
  • Multiclass = one of >2 exclusive; multilabel = several true at once. One-vs-all = k; one-vs-one = \(k(k-1)/2\). Binary relevance ≠ either.
  • Resample only TRAIN; test stays untouched. After resampling, test accuracy usually drops vs majority baseline.
  • AUC threshold-independent; accuracy/precision/recall/F1 are threshold-dependent. AUC=0.5 random; PR > ROC under imbalance.
  • Adjusted \(R^2\) can decrease; plain \(R^2\) only increases. CV is pessimistically biased (not optimistic).
  • WoE/IV after the split (uses target). "Too good to be true" → leakage.
  • Softmax ≈ argmax (not max). ReLU only hidden layers; output = sigmoid/softmax. Multilabel = per-output sigmoid.
  • Linear autoencoder ≈ PCA but NO orthogonality / variance ordering.
  • VAE: regularize BOTH mean and variance, else it acts like a plain AE.
  • t-SNE cluster size & inter-cluster distance are meaningless; not a clustering technique; non-parametric (can't embed new points).
  • Confidence = Supp(X∪Y)/Supp(X) ≈ \(P(Y\mid X)\) — don't divide by Supp(Y). Lift=1 → independence.
  • Apriori downward closure: subsets of frequent are frequent; supersets of infrequent are infrequent.
  • Self-attention mixes between tokens; FFN processes each independently. GPT = decoder-only, masked.
  • TF divides by document length \(|d|\), not #documents. IDF has the leading 1+.
  • MapReduce naive averaging is wrong — carry (avg, count) for re-reducible partial results.
  • HDFS is storage, not computing. Block = 64MB. Spark kicks out MapReduce via a DAG engine.
  • Graph DBs increase relational modeling (not "all NoSQL drops relations"). Transductive GNN eval leaks via message passing.
  • Girvan–Newman removes the highest-betweenness EDGE (not node); it is divisive. Louvain = greedy modularity.
  • Isolation forest: outliers = SHORT path length, random splits (not entropy).
  • Predicting the future ≠ extrapolating from training data (Olympic crossing lines).
