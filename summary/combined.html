01 · Analytics & Big Data Foundations

Framing

What data science actually is, the vocabulary, the learning taxonomy, process methodologies, and a critical look at "big data".

Core thesis

Analytics is not only machine learning, nor only big data. It is extracting valid, useful, unexpected, understandable knowledge from data for a purpose — and that purpose is very often automation, not ML for its own sake.

Must-know definitions

Three components of data science
Data + an algorithm + a purpose → discover patterns/models from data.
Four quality criteria
Valid (generalizable), Useful (actionable), Unexpected (interesting), Understandable (interpretable).
Instance / feature / target
Row/observation · input column (attribute, predictor, covariate, regressor) · output column (label, class, dependent variable). Binary is a special case of nominal categorical.
Featurization
Turning non-tabular data (text, image, audio) into a tabular feature matrix. Deep learning can also use data "as-is", though it still imposes some structure.

The learning taxonomy

AspectSupervisedUnsupervised
Target variableRequiredNot necessary
GoalRelate predictors to target (\(X \xrightarrow{f} Y\))Find structure/patterns
ExamplesChurn, fraud, credit risk, responseClustering, association rules, anomaly detection, dim. reduction
BasisFunction / mappingMostly frequency / distance (similarity)

Classification vs Regression

Categorical label (binary, multiclass, ordinal, multilabel) vs continuous label. Binary classification dominates tabular business settings. Don't swap multiclass (one of \(\gt 2\) mutually exclusive) with multilabel (several labels true at once).

Discriminative vs Generative

Discriminative models \(P(\text{label}\mid\text{data})\) directly as a function. Generative models the joint density and inverts via Bayes' rule.

Bayes' rule — the generative-model mechanism\[ P(\text{label}\mid\text{data}) = \frac{P(\text{data}\mid\text{label})\,P(\text{label})}{P(\text{data})} \]
Parametric vs non-parametric

Parametric: fixed functional form assumed (e.g. linear regression), parameters estimated (OLS, gradient descent). Non-parametric / data-driven: no fixed form — the form is learned (decision trees, ensembles, k-NN). "Non-parametric" does not mean "no parameters".

Process methodologies

Big Data — the V's, viewed critically

  • Volume, Velocity, Variety, Veracity + a fifth V = Value. ~90% of generated data is "unstructured".
  • Orthogonality: you don't need big data to do analytics, and big data doesn't necessarily mean you're doing analytics.
  • ML is about generalizable correlations, not causation (the Tanks / clever-Hans cautionary tale — the model learned weather, not tanks).
Exam watch-outs
  • Unsupervised needs no target; supervised requires one.
  • "Increasing model performance does not necessarily translate into business value" — and most business ML is really about automation.
  • Know AlexNet (2012, ~60M params, kickstarter of DL; Krizhevsky, Sutskever, Hinton) and Watson (Jeopardy! 2011).
  • A data scientist is not a magic unicorn — a multidisciplinary team is needed.

02 · Supervised I — Preprocessing & White-Box Models

Pipeline

Turning raw data into a clean feature matrix, then the three foundational interpretable techniques: (logistic) regression, decision/regression trees, k-NN.

The pipeline mantra

Everything you do over multiple instances (imputation rules, WoE, scaling, grouping) becomes part of the model pipeline. Fit on training data only; re-apply identically to validation/test/production. Never recompute a "new median" on test data. Set aside a hold-out test set as soon as possible, and beware future variables / leakage.

Preprocessing essentials

Missing values
Delete / impute (median for numeric — robust — and mode for categorical) / keep with a missing-value indicator. Distinguish not applicable from unknown.
Outliers
Invalid → treat as missing; valid → keep, truncate (cap, often at \(\mu \pm 3\sigma\)), or categorize. Trees are robust to outliers.
Standardization vs Normalization
Useless for decision trees; needed for distance-based methods.

Standardization

\[ x_{\text{new}} = \frac{x - \mu}{\sigma} \]

Mean 0, sd 1, unbounded. For Gaussian data & distance-based methods (SVM, k-NN, NN).

Normalization (scaling)

\[ x_{\text{new}} = \frac{x - x_{\min}}{x_{\max} - x_{\min}} \]

Bounded \([0,1]\); sensitive to outliers (min/max).

Encoding categoricals

  • Integer encoding wrongly imposes order/equal spacing on nominal categories.
  • Dummy / one-hot (\(k\) levels → \(k\), drop one → \(k-1\) baseline) avoids the dummy-variable trap in regression.
  • Odds-based grouping & Weights of Evidence use the target — must be done after the train/test split.
Weights of Evidence (monotonic with target)\[ \mathrm{WoE}_{\text{cat}} = \ln\!\left(\frac{p_{c_1,\text{cat}}}{p_{c_2,\text{cat}}}\right), \qquad \mathrm{IV} = \sum (p_G - p_B)\,\mathrm{WoE} \]

Information Value screens variables — important if \(\mathrm{IV} \gt 0.1\). Laplace smoothing pushes WoE toward 0 for small categories.

Linear & logistic regression

OLS — minimize SSE (convex, unique closed form if \(X\) full rank)\[ \arg\min_{\beta} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2 \]
Logistic regression — sigmoid bounds output to \((0,1)\)\[ P(y=1\mid x) = \frac{1}{1 + e^{-(\beta_0 + \beta_1 x_1 + \cdots)}}, \qquad \ln\!\frac{P(\text{yes})}{P(\text{no})} = \beta_0 + \sum_i \beta_i x_i \]
Logit interpretation

\(e^{\beta_i}\) = the odds-ratio: the multiplicative change in odds when \(x_i\) increases by 1. \(\beta_i \gt 0 \Rightarrow\) odds increase. Loss = cross-entropy \(-\,[\,y\log p + (1-y)\log(1-p)\,]\). Logistic regression is a classification technique (binary target) with a linear decision boundary.

Regularization

Lasso (L1, sparse) & Ridge (L2, shrink)\[ \arg\min_{\beta} \sum_i (y_i - \hat{y}_i)^2 + \lambda \sum_{j} |\beta_j| \quad\Big|\quad +\,\lambda \sum_{j} \beta_j^2 \]
Lasso (L1)Ridge (L2)
Coefficientssome exactly 0 (selection)shrink, rarely 0
Constraint shapediamondcircle
Correlated varspicks one at randomhandles better

Elastic net combines both; pick \(\lambda\) by cross-validation; normalize variables first so \(\lambda\) affects all coefficients equally. No penalty on the intercept.

Decision trees

Recursive partitioning via three decisions (split / stop / assign). Impurity measures:

Entropy · Information Gain\[ H(S) = -\sum_{x} p(x)\log_2 p(x), \qquad \mathrm{IG}(A,S) = H(S) - \sum_{t} p(t)\,H(t) \]
Gini · Classification error\[ \mathrm{Gini}(S) = 1 - \sum_{x} p(x)^2, \qquad \mathrm{ClassErr}(S) = 1 - \max_x p(x) \]
  • ID3 (categorical only, greedy, overfits) → C4.5 (continuous vars + pruning, info gain) → CART (regression trees).
  • Continuous & categorical features → binary splits (info gain is biased toward more subsets; binary avoids it and lets attributes be reused).
  • Prevent overfitting via early stopping or pruning (cost-complexity). Probabilities from pruned leaves are not well-calibrated.
  • Regression trees minimize within-node SSE (variance); predictions are piecewise-constant step functions.

Logistic regression

Single linear/diagonal boundary, smooth probability gradient, well-calibrated; needs scaling & dummies; sensitive to outliers.

Decision tree

Axis-aligned rectangles; captures non-linearity/interactions by default but struggles with linear relationships (step-function approximation); robust to outliers; unstable.

k-NN — "the data is the model"

Non-parametric, lazy learner. Find \(k\) nearest neighbours by distance → majority vote (classification) or distance-weighted average (regression). Requires normalization. No summarizing model; slow; unstable w.r.t. noise and the choice of \(k\). Good baseline / soundness check.

2

You compute Weights-of-Evidence and Information Value to encode and screen a categorical predictor. When must this be done?

  1. After the train/test split, fit on training data only — because WoE/IV use the target.
  2. Before any split, on the full dataset, since encodings are deterministic.

03 · Supervised II — Evaluation & Interpretability

It's all about generalization

How to know a model is good, and why it predicts what it predicts.

Central thesis

Do anything you want on the development (train) set, but evaluate on a strict, held-out test set made as challenging as possible. Goal = catch failure cases. 100% train accuracy is trivial (full tree, 1-NN, DB lookup). Split first, stratified, preferably out-of-time.

Confusion matrix & threshold metrics

The core denominators (memorize)\[ \text{Recall (TPR)} = \frac{TP}{TP+FN}, \quad \text{Precision} = \frac{TP}{TP+FP}, \quad \text{Specificity} = \frac{TN}{TN+FP} \]
F1 & balanced accuracy\[ F_1 = \frac{2\,P\,R}{P+R}, \qquad \text{Bal. Acc} = \frac{\text{recall}+\text{specificity}}{2} \]

FP = Type I error, FN = Type II error. All confusion-matrix metrics depend on the threshold; AUC, PR-AUC, CAP/AR, KS, h-measure, Mahalanobis do not.

Never trust accuracy alone

With a 1% positive rate, "always predict negative" gives 99% accuracy and zero positives caught.

Cost-sensitive learning & sampling

Cost-sensitive threshold\[ T_{CS} = \frac{C(1,0) - C(0,0)}{\big(C(1,0)-C(0,0)\big) + \big(C(0,1)-C(1,1)\big)} \]

Classify positive iff \(l(x,1) \lt l(x,0)\) where \(l(x,j) = \sum_k p(k\mid x)\,C(j,k)\). Symmetric unit costs \(\Rightarrow T_{CS}=0.5\).

Sampling ⇔ cost-sensitivity

Balancing classes by sampling is equivalent to an inverse-class-imbalance cost matrix. Resample only the training set; the test set stays untouched. After resampling, test accuracy usually drops below the always-majority baseline (you trade FPs for TPs). SMOTE interpolates between a minority point and its \(k\) nearest neighbours — it cannot invent future positives and is often not worth it for tree ensembles / deep learning.

Threshold-independent metrics

  • AUC = \(P(\text{random positive scores} \gt \text{random negative})\); coin flip = 0.5; threshold-independent.
  • PR curve > ROC under imbalance: precision ignores TNs, whereas FPR is inflated (made small) by dominant TNs.
  • CAP / Accuracy Ratio: \(\mathrm{AR} = 2\cdot\mathrm{AUC} - 1\) (the Gini). Lift of 1 = random. h-measure = cost-coherent AUC alternative (Hand 2009).
  • KS \(= \max_s |P(s\mid y{=}1) - P(s\mid y{=}0)|\).

Regression metrics & cross-validation

\(r^2\) and adjusted \(r^2\)\[ r^2 = 1 - \frac{\mathrm{SSE}}{S_{yy}}, \qquad r^2_a = 1 - (1-r^2)\frac{n-1}{n-k} \]
  • Adjusted \(r^2\) penalizes predictors and can decrease; plain \(r^2\) only ever increases. AIC/BIC trade fit vs complexity.
  • k-fold CV is for tuning; the test set is for true evaluation. Compare configurations by the average across folds (not the best fold). Fit preprocessing per-fold.
  • CV gives a pessimistically biased estimate; LOOCV is approximately unbiased. Bias–variance: complexity ↑ → bias² ↓, variance ↑.
  • Multiclass → one-vs-all (\(k\) classifiers, highest posterior) or one-vs-one (\(k(k-1)/2\), majority vote). Multilabel → binary relevance / label powerset.

Exam watch-outs
  • A Shapley value is the contribution to this output for this instance — not the change from removing the feature.
  • PDP can miss interactions (ceteris paribus) — "absence of evidence ≠ evidence of absence"; inspect data AND PDP.

04 · Supervised III — Ensembles & Modern Tabular

What wins on tabular

Bagging, boosting, the XGBoost family, and whether deep learning beats trees (mostly not).

Combining models

  • Combination rules (min/max/mean of probabilities) each need a new cutoff. Voting is less sensitive to probability distributions.
  • Mixture of experts (1991): gating network (EM-trained) weights experts: \(H = \sum_i w_i\,h_i(x,\theta_i)\). Stacking (1992): Tier-2 meta-classifier on Tier-1 outputs.

Classic random-forest traps

Two randomness sources: bootstrap of rows AND a random feature subset at every split — not once per tree. Heuristic \(M\): \(\lfloor N/3\rfloor\) for regression, \(\lfloor\sqrt{N}\rfloor\) for classification. Bootstrap 100% still leaves out \(\approx e^{-1}\approx 37\%\) (the OOB set). More trees never overfit; OOB is a check but not a substitute for a test set.

AdaBoost & gradient boosting

AdaBoost — learner weight & output (labels \(\pm 1\))\[ \alpha_t = \tfrac{1}{2}\ln\!\frac{1-\varepsilon_t}{\varepsilon_t}, \qquad H(x) = \mathrm{sign}\!\left(\sum_{t} \alpha_t\,h_t(x)\right) \]

Stops if \(\varepsilon_t \gt 0.5\); minimizes an exponential loss (additive-logistic view).

Gradient boosting — residual = negative MSE gradient\[ -\frac{\partial L_{\mathrm{MSE}}}{\partial F} = \tfrac{2}{n}\big(y - F(x)\big) \]

Gradient boosting fits the negative gradient of the loss w.r.t. the raw output; residual fitting is the special case for MSE. XGBoost adds 2nd-order info (gradient + Hessian) and L1/L2 regularization (loss + tree complexity).

Deep learning for tabular?

"In most cases: no(t really)." No spatial/sequential prior, limited data, trees capture interactions for free — gradient boosting wins honest benchmarks. Emerging exception: TabPFN (a Transformer pretrained on millions of synthetic tasks doing in-context Bayesian inference, no tuning; v2/v2.5 rival GBMs).

05 · Deep Learning I — Foundations & Images

CNNs · Autoencoders

From the perceptron and backprop to CNNs, interpretability, and generative models for images.

ANN foundations

Perceptron · gradient-descent update\[ o(X) = \sigma\!\left(\sum_i w_i x_i + w_b\right), \qquad w_{i,t+1} = w_{i,t} - \lambda\,\frac{\partial L}{\partial w_i} \]
  • A single perceptron cannot solve XOR (non-linearly separable); an MLP can. "Deep" = layers of representation, not deeper understanding.
  • Backprop = chain rule applied layer by layer. Universal approximation: 1 hidden layer suffices in theory (but depth is needed in practice).
  • One epoch = full pass over data; one iteration = single (mini-)batch update.
Vanishing gradients

Sigmoid/tanh derivatives are \(\lt 1\) and shrink toward the input layers — first-layer features matter most. Fixes: better activations (ReLU = \(\max(0,\text{sum})\), constant gradient, hidden layers only; can "die") and better initialization (He for ReLU \(2/\text{fan\_in}\); Glorot for tanh/sigmoid \(1/\text{fan\_avg}\); LeCun for SELU).

Softmax (smooth argmax, NOT max) · focal loss\[ \sigma(z)_i = \frac{e^{-\beta z_i}}{\sum_j e^{-\beta z_j}}, \qquad \mathrm{FL}(p_t) = -(1-p_t)^{\gamma}\log(p_t) \]

Multiclass → softmax output; multilabel → per-output sigmoid. Focal loss down-weights easy examples (\(\gamma\)).

Regularizing deep nets

  • Dropout: zero random nodes during training (different set each pass), scale activations down at test. Acts like an ensemble. Validation loss can look better than training loss because validation runs without dropout.
  • Batch normalization: normalize hidden activations per batch (internal covariate shift) → higher LR, slight regularization.
  • Data augmentation & transfer learning (freeze generic lower layers, retrain top) reduce data needs.

The convolutional architecture

Conv output size (W input, F filter, P padding, S stride)\[ \text{output} = \frac{W - F + 2P}{S} + 1 \]
  • Weight/parameter sharing (same kernel across all positions) is the key efficiency. Kernel depth = #input channels; #filters = output depth. Pooling (max/avg/global) keeps the same depth.
  • Families: LeNet → AlexNet (GPU, dropout, ReLU) → VGG (depth, ~140M params) → GoogLeNet (Inception) → ResNet (skip connections \(F(x)+x\) fight vanishing gradients; no final FC layers) → SqueezeNet (50× fewer params).
  • YOLO grids + IOU loss; ViT = pure transformer on image patches (CNNs not necessary). Adversarial examples fool any classifier. CNNs are biased toward texture, not shape.

Generative models for images

Autoencoder vs VAE vs GAN vs Diffusion

Autoencoder: encoder → bottleneck → decoder; output = input; reconstruction error for anomaly detection. A linear autoencoder ≈ PCA but without orthogonality/variance-ordering constraints.

VAE: encodes a distribution \(p(z\mid x)\), samples \(z\), regularizes with KL to \(\mathcal{N}(0,I)\) — must regularize both mean and variance.

GAN: generator vs discriminator; hard to train (no convergence guarantee, mode collapse, vanishing gradients). WGAN uses Earth-Mover's distance. Diffusion: add Gaussian noise forward, learn to denoise; often U-Net based; Stable Diffusion runs in a pretrained autoencoder's latent space.

VAE loss\[ \mathcal{L} = \lVert x - \hat{x}\rVert^2 + \mathrm{KL}\big[\mathcal{N}(\mu_x,\sigma_x)\,\big\|\,\mathcal{N}(0,I)\big] \]

06 · Unsupervised Learning

Cluster · Reduce · Detect · Associate

Clustering, dimensionality reduction, anomaly detection, association rules.

Clustering

Hierarchical

Agglomerative (bottom-up; all real implementations) vs divisive (top-down, expensive). No need to fix \(k\) — cut the dendrogram. Linkage: single (chaining), complete (tight), average (robust), centroid, Ward's (SSE increase, ANOVA-like).

Partitional (k-means)

Must specify \(k\) (elbow). Heuristic, local optima → k-means++ spreads centers (prob. \(\propto d(x,c)^2\)). Needs a mean (not categoricals), normalization; struggles with non-convex shapes/densities.

k-means objective (minimize distortion)\[ \min \sum_{k=1}^{K}\sum_{i=1}^{n_k} \lVert x_{ki} - \mu_k \rVert^2 \]
  • DBSCAN (Eps, MinPts; Core/Border/Noise) handles non-convex shapes and finds outliers natively.
  • EM/GMM = soft (probabilistic) assignment vs k-means' hard; learns a Gaussian per cluster (generative).
  • Davies–Bouldin Index: lower is better (low within-spread \(S_i\), high between-separation \(D_{ij}\)).

t-SNE — minimize KL divergence (Gaussian high-dim, Student-t low-dim)\[ \mathrm{KL}(P\|Q) = \sum_{i\neq j} p_{ij}\log\frac{p_{ij}}{q_{ij}} \]
t-SNE traps

t-SNE is not a clustering technique. In a t-SNE plot, cluster size and inter-cluster distance are not meaningful, and random data can look structured. Step 1 uses a Gaussian (perplexity sets \(\sigma_i\)); step 2 uses a Student-t (fatter tails, no perplexity).

Anomaly detection

  • Isolation forest: random splits; outliers isolated with short path lengths.
  • LOF: k-NN density; lower local density than neighbours → outlier.
  • (V)AE reconstruction error; CADE (real \(y{=}0\) vs synthetic uniform \(y{=}1\) classifier); one-class SVM (PU learning).

Association rules

Support · Confidence (\(\approx P(Y\mid X)\)) · Lift\[ \mathrm{Supp}(X) = \frac{|\{t : X\subseteq t\}|}{|T|}, \quad \mathrm{Conf}(X{\Rightarrow}Y) = \frac{\mathrm{Supp}(X\cup Y)}{\mathrm{Supp}(X)}, \quad \mathrm{Lift} = \frac{\mathrm{Supp}(X\cup Y)}{\mathrm{Supp}(X)\,\mathrm{Supp}(Y)} \]

Lift = 1 → independence; \(\gt 1\) → dependence. Apriori: min support → frequent itemsets, then min confidence → rules. The downward-closure / anti-monotonicity property (subsets of frequent itemsets are frequent; supersets of infrequent are infrequent) enables "join and prune". Power set size \(= 2^{|I|} - 1\). Sequence mining (GSP) respects order.

07 · Deep Learning II — Text & Recurrency

NLP · Embeddings · Transformers

From bag-of-words to word2vec, RNNs, attention, and the Transformer.

Classical text featurization

TF-IDF (TF divides by document length \(|d|\), not #docs)\[ \mathrm{TF}(t,d) = \frac{f(t,d)}{|d|}, \quad \mathrm{IDF}(t) = 1 + \log\frac{|c|}{|\{d : t\in d\}|}, \quad \mathrm{TFIDF} = \mathrm{TF}\cdot\mathrm{IDF} \]

BM25 saturates term frequency (nonlinear) and normalizes length better — sometimes beats embedding models. N-grams capture order but explode the feature set.

Word embeddings

The canonical analogy\[ [\text{king}] - [\text{man}] + [\text{woman}] \approx [\text{queen}] \]

CBOW

Context → focus word. Averaging (smoothing) activation, ignores context order, needs less data.

Skip-gram (CSG)

Focus → context. Pass-through activation, better-quality vectors but needs more data; easier to implement.

Keep the hidden-layer weights after training (size = embedding dim). GloVe exploits global co-occurrence; fastText (Facebook) uses subword char n-grams → embeds out-of-vocabulary/misspelled words.

RNNs → attention

Plain RNN cell\[ a^{\langle t\rangle} = \tanh\!\big(W_{aa}\,a^{\langle t-1\rangle} + W_{ax}\,x^{\langle t\rangle} + b_a\big), \quad \hat{y}^{\langle t\rangle} = \mathrm{softmax}\big(W_{ya}\,a^{\langle t\rangle} + b_y\big) \]
  • Trained by BPTT (weight sharing across time). In practice RNNs look back only a few steps (vanishing gradients).
  • LSTM (1997) passes two values (cell state \(C_t\) + hidden \(h_t\)); GRU (2014) passes one, trains faster, better on less data.
  • seq2seq: plain passes only the last encoder state; attention passes all states + per-step scoring.

The Transformer

Scaled dot-product self-attention (Q, K, V from the same set)\[ Z = \mathrm{softmax}\!\left(\frac{Q K^{\top}}{\sqrt{d_k}}\right) V \]
Exam watch-outs
  • Self-attention mixes information between tokens; the feed-forward net processes each token independently.
  • GPT = decoder-only with masked self-attention (mask later tokens → enables generation).
  • Positional encoding is added to embeddings (self-attention has no order). Multi-head = multiple Q/K/V sets, concatenated then \(\times W^O\).
  • Transformer generation cost \(\approx L^2\); RNN inference scales linearly — the Transformer's bottleneck. SSM/Mamba and diffusion LLMs are non-transformer alternatives.

08 · Big Data Platforms & Technologies

Systems

Where and how to run analytics across scales — from in-memory to Hadoop, Spark and streaming.

Deflationary message

Most problems are not big-data problems. Big data = "how to store/manage huge, high-speed, unstructured data"; analytics = "how to obtain insights" — they are orthogonal. A 512GB-RAM server is often cheaper than an HDFS cluster. "Insights > infra."

The scale ladder

  • In-memory (pandas, scikit-learn, tidyverse) → on-GPU / on-disk (Dask, vaex, Polars, DuckDB — DAG + columnar) → distributed.
  • File formats: AVRO = row-based, schema included, fast to write → ETL. Parquet = columnar, fast to read/query. Arrow = in-memory columnar spec (no serialization cost); Feather = Arrow-backed on-disk, language-agnostic.
  • Tooling: uv for packaging; git for versioning; MLOps (Sculley: ML is the "high-interest credit card of technical debt"; models degrade silently → concept drift; push vs pull runtime; Docker isolation).

Hadoop

  • Four modules: HDFS (storage, replicated 64MB blocks, NameNode + DataNodes), MapReduce (computation), YARN (resources), Common.
  • Map: function over every item → new list. Reduce: function over a list → scalar. Shuffle sends same-key entries to the same worker.
The reduce associativity trap

Reduce must give correct results even when called repeatedly on partial results, and emit the same structure as map. Naive averaging is wrong (re-reducing \([150, 300]\) gives 225, not the true 200). Correct: carry (running average, count) pairs and do a weighted combine.

SQL-on-Hadoop: Hive (converts SQL → MapReduce; slow), HBase (first DB, NoSQL), Pig, Impala/Tez (no MapReduce). "Big does not mean fast." Today: cloud warehouses (Redshift, Snowflake, Clickhouse) — "don't do it yourself."

Spark

  • Kicks out MapReduce, keeping (optionally) YARN + HDFS. Replaces it with a DAG execution engine with in-memory computing → 10–100× faster.
  • RDD = resilient distributed dataset; fault tolerance via lineage of transformations. Transformations are lazy; only actions trigger execution. Data is not loaded on the client (unless .toPandas()).
  • SparkSQL/DataFrames (Catalyst optimizer) impose named-column structure on RDDs — since Spark 2.0 the primary interface.
8

What is the relationship between Spark and Hadoop's MapReduce?

  1. Spark is built on top of MapReduce and uses it for every job.
  2. Spark replaces HDFS but keeps MapReduce as its execution engine.
  3. Spark kicks out MapReduce, replacing it with an in-memory DAG execution engine, while still able to use YARN and HDFS.
  4. In Spark, RDD transformations execute eagerly and load all data onto the client.

Constructing, measuring, mining and learning from network data — and storing it.

Construction is the first design choice

Graph data is (semi-)structured with no direct feature-vector representation. The same use case yields many valid networks, and that choice influences all downstream results. Recommended simple start (Oskarsdottir 2018): undirected, unweighted, binary, single node type, with extra edges to densify.

Featurization & GNNs

  • Markov property + homophily ("guilt by association"): a node's outcome depends on first-order neighbours. Relational neighbor classifier: \(P(F\mid?) = \) fraction of fraud neighbours.
  • DeepWalk/node2vec: random walks + skip-gram (word2vec). node2vec adds a biased 2nd-order walk (\(p,q\)). Drawbacks: no parameter sharing, params grow linearly with #nodes, not inductive.
Message passing (≡ convolution on irregular domains)\[ x_i^{k} = \gamma^{k}\!\Big( x_i^{k-1},\; \mathrm{AGG}_{j\in N(i)}\, \phi^{k}\big(x_i^{k-1}, x_j^{k-1}, e_{j,i}\big)\Big) \]

message (\(\phi\)/gather) → aggregate (sum/mean) → update (\(\gamma\)). GCN uses \(\hat{A}=A+I\) with symmetric normalization; GraphSAGE is inductive (sample + aggregate + CONCAT + L2-normalize); GAT uses multi-head attention. Stacking layers widens the receptive field.

Validation leakage (central trap)

"Feature engineering after the train/test split" is violated when features come from the whole graph. Transductive evaluation leaks: masked nodes still message-pass into scored nodes (yet widely used). Inductive uses a separate graph, but cutting edges removes information.

9

Why does transductive evaluation of a GNN leak information, and how do graph databases relate to other NoSQL stores?

  1. Transductive evaluation is leak-free; graph DBs drop relations like all NoSQL stores.
  2. Masked test nodes still message-pass their info to scored nodes (leakage); graph DBs increase relational modeling rather than dropping relations.
  3. Leakage comes only from the loss function; graph DBs use ACID like relational databases by default.
  4. Inductive evaluation leaks more than transductive because it reuses the same graph.

Cross-cutting Themes

Tie it together
  • Generalization above all. Split first, evaluate once on a strict held-out (preferably out-of-time) test set; train accuracy is meaningless. The same anti-leakage discipline reappears in WoE (ch.2), CV preprocessing (ch.3), and GNN evaluation (ch.9).
  • The pipeline is the model. Any statistic computed over instances (imputation, scaling, WoE, encodings) must be fit on train and re-applied identically — never recomputed on test/production data.
  • Representation learning is the unifying idea. Embeddings — from categorical2vec and word2vec to node2vec and GNN message passing — all learn dense, similarity-aware vectors; PCA/t-SNE/UMAP/autoencoders are the unsupervised cousins.
  • Bias–variance & regularization recur everywhere. L1/L2 (regression), pruning/early stopping (trees), bagging vs boosting, dropout/batch-norm (deep nets) — all manage the same trade-off.
  • Right tool, not biggest tool. Binary classification dominates tabular; gradient boosting usually beats deep learning on tabular; most problems aren't big-data problems. Match the technique (and the platform) to the business question and the purpose — automation, not ML for its own sake.
  • Threshold-dependent vs independent, hard vs soft, parametric vs data-driven, ACID vs BASE — the course is full of clean dichotomies that make excellent true/false traps.
