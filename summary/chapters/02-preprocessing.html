Supervised Learning — Preprocessing & Basic Techniques

The front end of the modeling pipeline: cleaning, encoding, feature engineering, and the three white-box learners — (logistic) regression, decision/regression trees, k-NN.

⏱️ ~18 min Pipeline fit on train only Encoding WoE · IV · dummies Regression OLS · logistic · L1/L2 Trees entropy · Gini · CART k-NN lazy learner

Big picture & the pipeline mindset

Supervised learning correlates features to a known target on a labelled dataset. The goal of preprocessing is a single feature matrix (instances × features + target) — most tabular techniques assume this single-table input.

Classification vs regression
Classification = categorical target (binary / multiclass / ordinal / multilabel). Regression = continuous target (absolute / delta / quantile).
Survival analysis
Instead of yes/no, predict the time until an event occurs.
◆ Why the pipeline mindset matters

Everything you do over multiple instances becomes part of the full model pipeline. Any transformation that uses statistics across instances (imputation rules, scaling parameters, WoE/grouping rules) must be fit on training data only and re-applied identically to validation / test / production data. The finished model cannot look into the future.

⚠ Exam watch-out — leakage / "future variables"

Do not include predictors known only after the target outcome, or that are "too perfectly" correlated with it. Ask: at the time the model is used, do you know this variable before or after the target? Set aside a hold-out test set as soon as possible.

Data types & EDA

  • Master data: core entities (Customers, Products…). Transactional data: events (POS, transfers) — needs a featurization step. External / open data: social, macro, weather, government (Eurostat).
  • Metadata = data about data (definitions, DBMS catalog).
▣ Anscombe's Quartet

Four datasets sharing near-identical summary stats — mean \(x=9\), var \(x=11\), mean \(y=7.50\), \(r=0.816\), fit \(y=3.00+0.500x\) — yet completely different when plotted. Lesson: summary statistics mislead → always visualize.

Missing values

Two reasons a value is missing
Not applicable (credit limit = NA when no card) vs unknown / not disclosed (age = NA). Also MAR/MCAR (missing at random) vs MNAR (missing not at random).

Three treatment options

  • Delete the row/column.
  • Replace (impute): mean, median (more robust), or mode (categoricals); or a model (often not worth it). Advanced: k-NN imputation, missForest.
  • Keep: add a separate missing-value indicator flag.

Common recipe: drop rows with too many missings, impute median (numeric) / mode (categorical), and add an "was-missing" column.

⚠ Exam watch-out — production imputation

Don't impute with a new median! New unseen instances also have missing values — use the same imputation rules derived from training data, applied identically to validation and production data.

Outliers & duplicates

Outliers are extreme observations (age = 241), and can be valid or invalid. Detect via histograms, z-scores, box-plots, violin plots.

  • Invalid outlier → treat as missing.
  • Valid outlier → depends on the technique: keep, truncate (cap) (often at \(\mu \pm 3\sigma\)), or categorize.

Anomaly-detection algorithms: Robust covariance, One-Class SVM, Isolation Forest, Local Outlier Factor. Duplicate rows can also be valid or invalid — treat accordingly.

Scaling: standardization vs normalization

Standardization → mean 0, sd 1\[ x_{new} = \frac{x - \mu}{\sigma} \]
Normalization (feature scaling) → bounded\[ x_{new} = \frac{x - x_{min}}{x_{max} - x_{min}} \]

⚠ Exam watch-out

Scaling is necessary for SVM, k-NN, neural networks (anything using Euclidean distance/similarity) and for regularized regression (so \(\lambda\) affects all variables similarly). Plain unregularized OLS does not need it for correctness. It is useless for decision trees.

Categorization (binning & grouping)

Also called coarse classification / classing / binning. Continuous → nominal (binning) or nominal → reduced nominal (grouping when too many levels).

◆ Why categorize

To treat outliers, improve interpretability, reduce the curse of dimensionality, and — crucially — introduce non-linear effects into linear models (e.g. a non-monotonic default-risk-vs-age curve a straight line can't capture).

⚠ Exam watch-out

Integer encoding of nominal categories is problematic — it imposes a false ordering and equal spacing the model reads as magnitude. Drop one dummy in regression to avoid the dummy-variable trap (the dropped level = reference baseline).

Odds-based grouping

Equal-width/frequency binning ignores the target. Instead: build a pivot table (attribute × target), compute odds = good/bad, and group categories with similar odds. Compare grouping options with the chi-square statistic.

Example odds (Purpose vs good/bad)\[ \text{car}=\tfrac{1000}{500}=2,\quad \text{cash}=\tfrac{2000}{100}=20,\quad \text{travel}=15,\quad \text{house}=6.25 \]

Weights of Evidence (WoE) & Information Value

WoE of a category\[ \text{WoE}_{cat} = \ln\!\left( \frac{p_{c1,cat}}{p_{c2,cat}} \right) \]

where \(p_{c1,cat}\) = share of all class-1 instances falling in this category, \(p_{c2,cat}\) = its share of all class-2 instances.

Key property
WoE has a monotonic relationship with the target; higher WoE = less risk when class 1 = good/non-event (the sign flips if the class coding is reversed). \(p_{c1}\gt p_{c2}\Rightarrow\) WoE \(\gt 0\).
Information Value — variable screening\[ \text{IV} = \sum (p_G - p_B)\cdot \text{WoE} \]

Important variables have IV \(\gt 0.1\). Choose category boundaries to maximize IV (use a sensitivity-analysis elbow on IV vs #categories).

Laplace smoothing\[ \text{WoE}_{cat} = \ln\!\left( \frac{p_{c1,cat} + n}{p_{c2,cat} + n} \right) \]

Larger \(n\) → less reliance on data → pushes WoE toward 0. Fewer instances in a category → less reliable/stable WoE.

⚠ Exam watch-out — target leakage

WoE and IV use information from the target, so they must be computed after the train/test split (on training data only), otherwise you leak the target.

High-cardinality categoricals

Dummies don't solve a categorical with too many levels (dimensionality explodes). Options: domain grouping (NACE codes); frequency-based grouping (e.g. postal_code_count = how often that code appears in training — loses detail but captures frequency, keep the same rule for production); odds/WoE grouping; decision-tree-based; embeddings (entity embeddings for high-cardinality, similar values map close together).

Feature engineering

◆ Definition & motivation

"Transform dataset variables into features so analytical models achieve better predictive performance, interpretability, or both." Take the bias of your technique into account — e.g. craft features to make data linearly separable for a linear model (concentric rings → separable in polar coordinates). Remember: correlation ≠ causation.

RFM & recency decay

RFM = Recency, Frequency, Monetary value of transactions (popular in marketing & fraud).

Recency (exponential decay)\[ \text{recency} = e^{-\gamma t} \]

Larger \(\gamma\) → faster decay. Never occurred → recency \(=0\); very recent → recency \(\approx 1\).

Time & date features

⚠ Exam watch-out — time is circular

\(00{:}00 = 24{:}00\) and there's no natural ordering (is 23:00 \(\gt\) 01:00?). Do not use the arithmetic mean for timestamps. Model time periodically with the von Mises distribution: \(\mu\) = periodic mean, and \(1/\kappa\) = periodic variance.

From dates derive: time-since, quarter, month, week, weekend/holiday flags, and sin/cos transforms for cyclic encoding.

Deltas / trends & ordinal coding

Trends over time\[ \text{absolute}=\frac{F_t - F_{t-x}}{x}, \qquad \text{relative}=\frac{F_t - F_{t-x}}{F_{t-x}} \]

Beware denominators = 0! For ordinal features, use percentile coding (cumulative percentiles) or thermometer coding (each higher level fills one more flag, preserving order) — not plain integer coding.

Featurization & selection

Featurization = squeeze a representational vector out of unstructured data (deep learning as feature extractor; tools: featuretools, tsfresh, stumpy, skrub). Feature selection: low variance, chi-square, information gain, correlation, IV, stepwise, regularization, variable importance, PCA. Lots of features is fine only if you have far more instances.

Linear & logistic regression

Linear regression (OLS)

Model\[ y = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + \dots + \varepsilon, \quad \varepsilon \sim N(0,\sigma^2) \]
Ordinary Least Squares — minimize SSE\[ \arg\min_{\beta} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2 \]

SSE/MSE is convex (single minimum, fully differentiable); if \(X\) is full rank there is a unique closed-form solution. Gradient descent may be preferred at scale.

Why not linear regression for classification?

◆ Motivation

A linear output gives no guarantee of being in [0,1] (or even 0/1), and the binary target/errors are not normally distributed → OLS assumptions violated. Solution: bound the output with the sigmoid.

Logistic / sigmoid\[ f(z) = \frac{1}{1 + e^{-z}} \]
Logit = log-odds is linear\[ \ln\!\left( \frac{P(\text{yes}\mid\cdot)}{P(\text{no}\mid\cdot)} \right) = \beta_0 + \beta_1 x_1 + \dots \]
⚠ Exam watch-out

Logistic regression is a classification technique, not regression (the target is binary categorical), even though its predictions are continuous in \(]0,1[\) — a probability.

Coefficient interpretation

Odds-ratio: effect of +1 in \(X_i\)\[ \text{odds}\big|_{X_i+1} = \text{odds}\big|_{X_i}\cdot e^{\beta_i}, \qquad e^{\beta_i} = \text{odds-ratio} \]
  • \(\beta_i \gt 0 \Rightarrow e^{\beta_i} \gt 1 \Rightarrow\) odds/probability increase with \(X_i\).
  • Doubling amount (change in \(X_i\) to double the odds): \( \dfrac{\log 2}{\beta_i} \).
Loss — cross-entropy / log loss (binary)\[ -\big( y\log p + (1-y)\log(1-p) \big) \]

Logistic regression is: a formula (easy to interpret, usually well-calibrated when correctly specified), with a linear decision boundary, but sensitive to outliers, correlated features, and the curse of dimensionality; categoricals must be encoded.

1

In a logistic model, a coefficient \(\beta_i = 0.69\) for feature \(X_i\) means a one-unit increase in \(X_i\)…

  1. adds 0.69 to the predicted probability
  2. adds 0.69 to the odds of the positive outcome
  3. multiplies the odds of the positive outcome by \(e^{0.69}\approx 2\)

Regularization: Lasso vs Ridge

◆ Motivation

A steeper-slope line can have lower SSE yet generalize worse. Penalize the size of the weights: constrained — not fewer — parameters, making the model less sensitive to outliers and better at generalizing. No penalization on the intercept.

Lasso (L1)\[ \arg\min_{\beta} \sum_i (y_i - \hat{y}_i)^2 + \lambda \sum_{j=1}^{p} \lvert \beta_j \rvert \]
Ridge (L2)\[ \arg\min_{\beta} \sum_i (y_i - \hat{y}_i)^2 + \lambda \sum_{j=1}^{p} \beta_j^{2} \]

Other regression variants (know the one-liners)

  • Stepwise selection — statistically illegitimate (biased \(R^2\), wrong test distributions, falsely narrow CIs); even AIC is a random variable. Use a proper train-validation-test setup.
  • Multinomial logistic (K classes) → builds \(K-1\) binary models vs a reference class. Ordinal logistic = "proportional odds" (parallel logits differing only in intercept).
  • PCR = PCA then regression (ignores target); PLS includes the target — but often behaves like PCR (variance dominates). Both need standardization.
  • OLS minimizes vertical residuals (x exact, y noisy); TLS/ODR minimizes orthogonal distance (both noisy) — but TLS can have worse vertical prediction error than OLS.

Decision trees

For (multi)classification and regression via recursive partitioning. Three decisions: splitting, stopping, assignment. Algorithms: ID3, C4.5, J48, CART.

Impurity measures

Entropy (uncertainty in bits)\[ H(S) = -\sum_{x \in X} p(x)\log_2 p(x) \]
Gini diversity\[ \text{Gini}(S) = 1 - \sum_{x \in X} p(x)^2 \]
Information gain of split on \(A\)\[ IG(A,S) = H(S) - \sum_{t \in T} p(t)\,H(t),\quad p(t)=\frac{\lvert t\rvert}{\lvert S\rvert} \]
MeasurePeak at \(p=0.5\)Notes
Entropy1.0bits; bounded 0–1 only for 2 classes
Gini0.5default in most libs; faster, better for continuous
Classification error \(1-\max p\)0.5piecewise-linear
⚠ Exam watch-out

\(H(S)=0\) means a completely pure set. Entropy is bounded in [0,1] only for two-class problems. Gini is the default, slightly faster, a bit better for continuous variables.

Algorithms & continuous splits

  • ID3 — categorical features only; greedy (no backtracking → global optimum not guaranteed); grows until it can't → overfits.
  • C4.5 — extension of ID3; main contribution = handling continuous variables; can handle missing (paper ignores them in IG, most implementations don't); describes pruning.
  • Continuous splits: enforce binary splits at thresholds (≤21, ≤24…), testing only distinct training values, picking the one maximizing IG.
⚠ Exam watch-out

Information gain is biased toward splits producing more subsets → always using binary splits removes this bias (and lets attributes be re-used down the tree; deeper but less wide).

Overfitting, pruning & probabilities

  • Early stopping: min instances per node, max depth, or a validation set whose accuracy starts to fall.
  • Pruning: grow full tree, then merge low-value leaves (cost-complexity / weakest-link), governed by a complexity parameter.
  • Fully-grown trees give no probabilities (pure leaves). After pruning/stopping, a leaf "8 A vs 2 B" → \(P(A\mid x)=80\%\) — usable to rank, but not well-calibrated and ignores leaf frequency by default.

Regression trees

Made popular by CART (Classification And Regression Trees). Predict the mean continuous label per leaf; split to minimize within-node variance / SSE (max between-node separation, like k-means). A regression tree typically needs to be deeper than a classification one; predictions are piecewise-constant step functions (a model tree fits a linear model per leaf).

Tree vs logistic regression

Logistic regression

  • Single linear (diagonal) boundary; smooth probability gradient
  • Captures linear relations naturally; needs explicit interaction features
  • Well-calibrated; needs scaling, dummies; sensitive to outliers

Decision tree

  • Axis-aligned rectangles ("squares orthogonal to a dimension")
  • Struggles with linear relations (best = step approximation); captures non-linearity/interactions by default
  • Not calibrated, unstable; robust to outliers, little prep needed

k-Nearest Neighbors (k-NN)

Definition
A non-parametric method for classification & regression. "The data is the model." Distance-based → normalization/standardization required.
How
Find the \(k\) closest training points; classify by majority vote, regress by a (weighted) average weighted by the inverse of distance.
⚠ Exam watch-out — lazy learner

k-NN is a lazy learner: it learns no summarizing model (no formula, no tree). You must keep the training data, prediction is slow, and it is unstable with respect to k and noisy data. Training cost ≈ none; prediction cost is high.

One-minute recap

  • Pipeline: fit all transforms (imputation, scaling, WoE, grouping) on training data only; re-apply identical rules to test/production. Never compute a new median on test data. Set aside a hold-out test set early.
  • Missing: delete / impute (median & mode) / keep + indicator. Outliers: invalid → missing; valid → keep / cap (\(\mu\pm3\sigma\)) / categorize.
  • Scaling: standardize \(\frac{x-\mu}{\sigma}\) (N(0,1)), normalize \(\frac{x-x_{min}}{x_{max}-x_{min}}\) (bounded) — needed for SVM/k-NN/NN/regularized regression, useless for trees.
  • Encoding: integer = false order (bad); one-hot drop one (regression); WoE = \(\ln(p_{c1}/p_{c2})\), monotonic with target, after the split; IV \(\gt 0.1\) screens variables.
  • Logistic: classification, output \(]0,1[\); \(e^{\beta_i}\) = odds-ratio; loss = cross-entropy.
  • Regularization: Lasso (L1) = diamond → exact zeros / selection, no grouping; Ridge (L2) = circle → shrink + grouping; no penalty on intercept; normalize first; \(\lambda\) via CV.
  • Trees: entropy/Gini (Gini default), IG biased to more subsets → binary splits; ID3 categorical+greedy+overfits, C4.5 adds continuous; CART = regression (mean per leaf, min variance); axis-aligned, robust to outliers, unstable, not calibrated.
  • k-NN: lazy ("data is the model"), distance-based → scale; slow prediction, unstable wrt k.
