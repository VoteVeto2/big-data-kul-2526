Deep Learning for Graphs

Graph construction, centrality, community mining, network-based inference, shallow embeddings, and GNNs (GCN / GraphSAGE / GAT) — plus validation traps and graph databases.

Big picture & why graphs are hard

This is the third Deep-Learning lecture (after CNNs and NLP/word2vec). It reuses word2vec/skip-gram (DeepWalk, node2vec), CNN convolutions (message passing = convolution on irregular domains), and PageRank.

Graph
Nodes (vertices / actors: people, pages, cards) connected by edges (links / interactions: friendship, transaction, "like").
◆ Why it's tricky

Graph data is (semi-)structured but has no direct feature-vector representation. The same use case can be turned into many valid networks, and the construction choice influences all downstream findings. Step one — deciding how and why to build the graph — is itself the hardest part.

Recurring example: credit-card↔merchant networks where fraudsters cluster (stolen cards used at the same stores).

Graph construction

Design choices that change everything

  • Nodes: homogeneous (one entity type) vs. heterogeneous (e.g. products + customers); may carry features and a single node weight (intensity of presence).
  • Edges: one type vs. many; weights are very common (vs. binary presence); directed / undirected / self / reciprocal / duplicated. An edge joining more than two nodes = a hypergraph.
  • Centerpoint: labels usually live on nodes. If the focus is on edges, re-encode an edge as a node = a line graph.
✓ Start-simple recipe (Oskarsdottir 2018)

Undirected, unweighted, binary, single node type, plus extra edge info (e.g. pseudo-edges) for a densely connected graph. Empirically: using all edges beats only-reciprocated edges (higher AUC & lift@10%), and undirected networks performed best.

Bipartite → unipartite projection

A bipartite graph (authors↔papers) can be projected to a unipartite one (author↔author). Projection edge weights can be binary, common-neighbours count, or Jaccard — e.g. \( w_{A,B} = \{1, 2, 0.66\} \) for {binary, common neighbours, Jaccard}.

Recency-weighted summary graph (temporal)

recency edge weight\[ f(h) = e^{-\alpha h} \]

\(h\) = age of the interaction; \(\alpha\) controls decay. Larger \(\alpha\) → faster decay (recent interactions dominate). At \(h=0\), \(w=1\).

Adjacency vs. incidence matrix

Adjacency matrix

node × node. Entry = 1 if an edge connects the two nodes.

Incidence matrix

node × edge. Entry = 1 if the node is an endpoint of that edge.

⚠ Exam watch-out

Do not swap them. Adjacency = node×node; incidence = node×edge.

Centrality (which nodes are important?)

Centrality finds the most important nodes (influencers, hubs, gatekeepers, super-spreaders). Their values can also be used as features downstream.

The four measures

ClosenessGeographicreciprocal of farness — inverse mean geodesic distance to allQuick reach, high visibility

Geodesic
shortest path between two nodes (computationally intensive for large graphs).
Graph-theoretic center
node(s) with the lowest maximum distance to all others: \( \arg\min_{n} \max_{m} \lvert \mathrm{geod}(n,m) \rvert \).
▸ Degree subtlety

A high-degree node is not necessarily the most "connected" entity — its many links may all point to low-level entities.

Eigenvector / PageRank

Connections to high-scoring nodes contribute more than equal connections to low-scoring ones. PageRank is a variant of eigenvector centrality for directed graphs. Computed by iterating a stochastic matrix \(M\) against a score vector until convergence — but real implementations use a "random walkers" approach to handle scale, loops, disconnected graphs and dangling links.

power iteration (slide 28)\[ \left[\tfrac{1}{3},\tfrac{1}{3},\tfrac{1}{3}\right] \to \left[\tfrac{1}{3},\tfrac{1}{2},\tfrac{1}{6}\right] \to \left[\tfrac{5}{12},\tfrac{1}{3},\tfrac{1}{4}\right] \to \cdots \to \left[\tfrac{2}{5},\tfrac{2}{5},\tfrac{1}{5}\right] \]
⚠ Kite-network trap (Krackhardt 1988)

No single node wins on all measures. Diane wins degree & PageRank; Fernando/Garth win closeness; Heather wins betweenness (the bridge to the Ike–Jane tail). Different measures crown different "kings".

⚠ Exam watch-out

Girvan–Newman removes the highest-betweenness edge (not node) and is divisive. Louvain is greedy/agglomerative (each node its own community first) and optimizes modularity.

Network-based inference (predictions)

◆ The setup

Labels live on nodes; we want to infer them for unknown nodes (fraud, churn, age). Comparable to semi-supervised learning with a few known positives — hard because nodes influence each other.

Markov property (graphs)
a node's outcome depends only on its first-order (1-hop) neighbours. Valid in many settings; tied to homophily.
Homophily
"birds of a feather flock together" / "guilt by association" — edges among same-class nodes are more common than between classes (fraudsters cluster; a customer linked to a recent churner is likelier to churn).

Relational Neighbor Classifier

Class probability = fraction of neighbours with that class (simple, "but boring"; uses the Markov property). With neighbours \(\{NF,F,F,NF,NF\}\), \(Z=5\):

relational neighbour\[ P(F\mid ?) = \tfrac{2}{5}, \qquad P(NF\mid ?) = \tfrac{3}{5} \]

Personalized PageRank

Models how information spreads. Uses "lazy random walks" (can rest at a vertex). Plain PageRank teleports uniformly; personalized PageRank teleports with a non-uniform distribution biased toward a class of interest (\(Y=1\)). The propagated scores become predictions — or features.

⚠ Exam watch-out

Personalized PageRank uses label information (biased teleport) → it leaks more than label-agnostic centrality metrics. Plain PageRank teleports uniformly.

Relational logistic regression (Lu & Getoor 2003)

Combines local attributes (age, income, RFM) with network attributes from neighbours in one logistic model:

  • mode-link = most frequent neighbour class
  • count-link = frequency of each neighbour class
  • binary-link = binary presence indicators per class

For neighbours \(\{NF,F,F,NF,NF\}\): mode = NF, freq(NF)=3, freq(F)=2, binary(NF)=1, binary(F)=1.

✓ Predictions-as-features

Relational-learner outputs (PPR scores), centrality metrics, community labels, and egonet/domain features can all be stacked into a normal classifier. Featurization summary: keep the network simple, use egonet domain features + Personalized PageRank as a "global network" feature.

Validation: data leakage is the central trap

▲ The core problem

The rule "all feature engineering happens after the train/test split" is violated when features come from the whole graph → data leakage. Neighbours of test nodes may be in train (and vice versa); labels and structure also change over time.

Transductive

Test on the same graph the model saw, with some nodes' labels masked during training. Easy — but masked nodes still message-pass their info to scored nodes → leakage (widely used in papers anyway).

Inductive

Test on a separate graph not seen in training. Requires GNN ops independent of #nodes/edges. But splitting one graph means cutting edges removes information; temporal splits with overlapping windows still leak.

Mitigations: make the time gap large, test at multiple points, randomly censor positive labels during feature generation, and prefer features that leak less (most centrality metrics use no labels; PPR uses labels). Key reference: Guo & vanden Broucke (2024), "A Critique on Transductive Evaluation for GNN Node Classification."

Shallow embeddings: DeepWalk & node2vec

Random walk
start at a vertex; each step move to a neighbour. Unweighted → uniform among neighbours; weighted → proportional to edge weight.

DeepWalk (Perozzi et al. 2014)

Unsupervised node embeddings: generate random walks, then train a skip-gram model (word2vec) where walks = sentences, nodes = words. Preserves structural properties in a low-dimensional space.

skip-gram objective\[ J(\Phi) = -\log \Pr\big(u_k \mid \Phi(v_j)\big), \qquad \Phi \leftarrow \Phi - \alpha \tfrac{\partial J}{\partial \Phi} \]

node2vec (Grover & Leskovec 2016)

Same idea + the biased 2nd-order random walk with parameters \(p\) (return) and \(q\) (in-out):

node2vec search bias\[ \alpha_{pq}(t,x) = \begin{cases} 1/p & d_{tx}=0 \;(\text{return}) \\ 1 & d_{tx}=1 \\ 1/q & d_{tx}=2 \end{cases} \]

Steering the walk gives BFS-like (structural) vs. DFS-like (community) representations.

⚠ Drawbacks (shallow embeddings)
  • No parameter sharing between nodes → params grow linearly with #nodes.
  • Hard to include node/edge features.
  • Not inductive — cannot handle dynamic graphs / new nodes.

Graph Neural Networks

◆ Goal

One general structure that learns node/edge/graph representations, generalizes to unseen elements, and uses both features and structure. Graphs carry four kinds of info: node, edge, global features, and connectivity.

Feeding a graph in

Features → a feature matrix (easy). Connectivity is harder: the adjacency matrix is sparse, and a DNN is not permutation invariant — many adjacency matrices encode the same graph but different orderings give different outputs. Adjacency lists (edge \(e_k\) stored as a tuple \((i,j)\)) are the memory-efficient, order-robust alternative.

Two problems → pooling & message passing

Problem 1 — task/object mismatch → Pooling
info in edges but predictions on nodes? Gather the relevant embeddings, then aggregate (sum/mean). Graph-level pooling resembles global average pooling at the end of a CNN.
Problem 2 — not using connectivity → Message passing
a plain GNN (just an MLP per node) ignores connectivity during learning. Message passing makes embeddings connectivity-aware: gather neighbour messages → aggregate → update.
message passing — generic\[ x_i^{k} = \gamma^{k}\Big( x_i^{k-1},\; \underset{j \in N(i)}{\mathrm{AGG}}\; \varphi^{k}\big(x_i^{k-1}, x_j^{k-1}, e_{j,i}\big) \Big) \]
  • \(\varphi\) = gather / message function
  • \(\mathrm{AGG}\) = permutation-invariant aggregator (sum/mean)
  • \(\gamma\) = update function (usually an MLP)
simplified — "looks like a convolution"\[ x_i^{k} = \gamma^{k}\Big( \sum_{j \in N(i)\cup\{i\}} x_j^{k-1}\, W_{j,i} \Big) \]
▸ Message passing = convolution

An image is a regular grid graph (pixels = nodes, neighbours = adjacent pixels); its adjacency matrix is banded; a CNN convolution is message passing on that grid. The simplified form's weakness is \(W_{j,i}\) (one weight per node pair) → we want a gather function \(\varphi\) with weight sharing.

Message passing is the generic approach (message → aggregate → update); spectral methods are a special case that does not aggregate over neighbours and avoids stacking many layers. "Localized" is not really a drawback — stacking GNN layers widens the receptive field (like CNN depth).

GCN (Kipf & Welling 2016)

GCN — one-operation matrix form\[ X' = \hat{D}^{-1/2}\,\hat{A}\,\hat{D}^{-1/2}\,X\,\Theta, \qquad \hat{A} = A + I \]

\(\hat{A} = A+I\) adds self-loops; \(\hat{D}\) is the diagonal degree matrix \(\hat{D}_{ii} = \sum_j \hat{A}_{ij}\); \(\Theta\) = learnable kernel. Symmetric normalization on both sides.

GraphSAGE (Hamilton et al. 2017) — inductive

An inductive framework that generates embeddings by sampling and aggregating a node's local neighbourhood:

GraphSAGE layer\[ h_v^{k} = \sigma\!\Big( W^{k}\cdot \mathrm{CONCAT}\big(h_v^{k-1},\; \mathrm{AGG}_k\{h_u^{k-1} : u \in N(v)\}\big)\Big),\quad h_v^{k} \leftarrow \frac{h_v^{k}}{\lVert h_v^{k}\rVert_2} \]

Key traits: samples neighbours, CONCATs self + neighbour embeddings, then L2-normalizes. (In PyG, the unit and the NeighborLoader are split.)

GAT (Veličković et al. 2017) — attention

Adds the attention mechanism into the propagation step: each node attends to its neighbours via self-attention. Coefficients \(\alpha_{ij}\) = softmax of a LeakyReLU score; supports multi-head attention (heads concatenated or averaged).

Learning tasks & pipeline

  • Node-level: classify/regress/cluster nodes (e.g. detect fake accounts).
  • Edge-level: predict an edge label or its existence = link prediction (will a user like a product?).
  • Graph-level: classify a whole graph (e.g. molecule toxicity from structure).

General GNN block: optional Sampling → Conv/Recurrent → optional Pooling operator, with optional skip connections; trained supervised / semi- / unsupervised.

Graph databases & Cypher

◆ The twist

A graph DB is a NoSQL database, but contrary to other NoSQL it increases relational modeling instead of dropping relations. It's a hyper-relational DB where JOIN tables become semantically meaningful relationships you navigate (traversal) and query by pattern matching. (Neo4j, Memgraph, FlockDB.)

ACID vs. BASE

ACID — Relational

Atomicity, Consistency, Isolation, Durability. Schema-driven, SQL, strong consistency.

BASE — NoSQL

Basically Available, Soft state, Eventual consistency. Schema-free, horizontally scalable.

Eventual consistency: replicas converge at some point after a transaction — continuous consistency is not guaranteed. ("NoSQL" was first used in 1998 for a relational DB that simply forewent SQL — unrelated to the modern "not only SQL" movement.)

Cypher (Neo4j's query language)

Declarative, SQL-like, but geared to patterns via MATCH. Nodes = (), type-filter with : as (b:Book); edges = -- (undirected) or --> (directed); relationship type in square brackets -[:WRITTEN_BY]-; transitive paths via * with min..max.

SQL — 3 tables + join

SELECT title FROM books, authors, books_authors
WHERE author.id = books_authors.author_id
  AND books.id = books_authors.book_id
  AND author.name = "Seppe"

Cypher — nodes + relationship

MATCH (b:Book)<-[:WRITTEN_BY]-(a:Author)
WHERE a.name = "Seppe"
RETURN b.title

Recursive/transitive hierarchy queries (-[:PARENT*0..]-) are easy in Cypher but hard in SQL even with recursive extensions.

⚠ Exam watch-out

Neo4j is optimized for OLTP and was not originally built for graph compute/analytics — that came later via the Neo4j GDS library (and Memgraph's MAGE). GraphX extends the Spark RDD as a directed multigraph; GraphFrames is DataFrame-based and "never really took off."

One-minute recap

  • Construction matters: same use case → many graphs; start simple (undirected, unweighted, binary, single type). Hypergraph = edges on >2 nodes. Adjacency = node×node, incidence = node×edge.
  • Centrality: degree (hub), closeness \((N-1)/\sum \mathrm{geod}\), betweenness (gatekeeper, also for edges), eigenvector/PageRank (directed). No node wins all (kite).
  • Communities: Girvan–Newman = divisive, removes highest-betweenness edge; Louvain = greedy, maximizes modularity; k-core = each vertex ≥ \(k\) links inside.
  • Inference: Markov = 1-hop neighbours; homophily = guilt by association; Personalized PageRank biases the teleport with labels (so it leaks).
  • Embeddings: DeepWalk = walks + skip-gram (unsupervised); node2vec adds biased walk (\(p,q\)). Shallow = no parameter sharing, not inductive.
  • GNN: message passing (φ message → AGG → γ update) = convolution on irregular graphs; stacking widens receptive field; spectral is a special case. GCN \(\hat A=A+I\); GraphSAGE = inductive sample+CONCAT+L2; GAT = attention.
  • Validation: whole-graph features leak; transductive masks labels but nodes still message-pass → leakage.
  • DBs: graph DB = NoSQL that adds relations; ACID vs. BASE; Cypher MATCH patterns; Neo4j = OLTP.
