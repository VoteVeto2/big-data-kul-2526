Deep Learning for Text

From bag-of-words and TF-IDF to word embeddings, RNNs, attention, and the decoder-only Transformer behind GPT — the exam-likely 20%.

Big picture: before vs. after deep learning

Just like computer vision, text splits into a "before and after deep learning" era = manual vs. automated feature engineering. The cross-cutting theme is representation learning: find a dense, low-dimensional, real-valued vector (an embedding) per item so that distances capture similarity — and learn it from data instead of hand-crafting it.

Why text is hard

Text is everywhere but unstructured / semi-structured: no natural feature-vector, has linguistic structure (word relationships, importance, negations), is dirty (spelling, abbreviations, homographs), and is meant for human-to-human communication. The deck's one-liner: "the main task of ML and especially deep learning is learning compressed, dense representations that fit the downstream task."

Corpus / corpora
A collection of documents. Each document is built from tokens / terms (words, punctuation, abbreviations, smileys).
Featurize
Turn a free-form sequence of words into a numeric feature vector — already an instance of representation learning.

Classical text featurization

Bag of words (BoW)

Treat each document as an unordered collection of tokens. One feature per vocabulary word; value = 1 (present) / 0 (absent). Ignores grammar, order, structure. Cheap, but causes an explosion of features.

Vocabulary = {this, is, an, a, example, second}:

  • "This is an example" → 1 1 1 0 1 0
  • "This is a second example" → 1 1 0 1 1 1

Cleaning: normalization, stemming, lemmatization, stop-words

Normalization
Lowercase every term so case does not split the vocabulary.
Stemming
Suffix-stripping to keep the root (plurals → singular). Porter's algorithm = basically suffix removal.
Lemmatization
More complex: maps to a real dictionary root handling irregulars (better → good, flies → fly). May need PoS tagging for context.
Stop-word removal
Drop very common words ("the", "and", "of", "on").

Term frequency & the document-term matrix

BoW with counts instead of binary presence → the document-term matrix. Raw counts are best normalized (e.g. divide by document length). Words should be not too common, not too rare → impose an upper and lower limit on how many documents a word may appear in.

TF-IDF

Why

Pure term frequency over-weights common words. TF-IDF balances local importance (TF) against global rarity (IDF) — terms should be neither too rare nor too common.

Term frequency — divide by document length \[ \mathrm{TF}(t_i, d) = \frac{f(t_i, d)}{\lvert d \rvert} \]
Inverse document frequency — note the leading 1+ \[ \mathrm{IDF}(t_i) = 1 + \log\!\left( \frac{\lvert c \rvert}{\lvert \{ d \in c : t_i \in d \} \rvert} \right) \]
TF-IDF weight per term per document \[ \mathrm{TFIDF}(t_i, d) = \mathrm{TF}(t_i, d) \times \mathrm{IDF}(t_i) \qquad d = [\,w_1, w_2, \ldots, w_m\,] \]
⚠ Exam watch-out — the TF denominator

TF divides by the document length \(\lvert d \rvert\), NOT by the number of documents. The number of documents \(\lvert c \rvert\) lives in the IDF numerator. IDF is inverse: large for rare terms, decaying toward 1 as a term appears in more documents.

Jazz worked example (slide 12)

15 jazz biographies from Wikipedia → ~2,000 features after stemming & stop-word removal. Query "Famous jazz saxophonist born in Kansas who played bebop and latin" → cosine similarity ranks Charlie Parker (0.135) top, then Count Basie (0.119), Dizzy Gillespie (0.086), John Coltrane (0.079). The term "latin" carries the highest per-term TF-IDF (~0.52).

BM25 (Okapi "Best Matching")

A ranking function for document relevance to a search query. Two improvements over TF-IDF: it saturates term frequency (repeating a word 100× is not 100× more important — nonlinear) and normalizes document length better.

BM25 per-term weight (params k₁, b) \[ w_{t_i,d} = \mathrm{IDF}(t_i)\cdot \frac{ f(t_i,d)\,(k_1+1) }{ f(t_i,d) + k_1\!\left(1 - b + b\,\dfrac{\lvert d \rvert}{\text{avgdl}}\right) } \]

Typical \(k_1 \in [1.2, 2.0]\), \(b = 0.75\); avgdl = average document length. Key claim: "works better than TF-IDF in many cases, and sometimes even beats embedding models."

N-grams & handling high dimensionality

N-grams use sequences of adjacent words as terms (adjacent pair = bi-gram), capturing word order that BoW ignores — but they greatly increase the feature set. To tame the high-dimensional sparse matrices: feature selection; fast scalable linear Naive Bayes / SVM; clustering via NMF; the hashing trick (collapse to \(n\) hashed features); and reduction with t-SNE / UMAP.

Classical NLP

Learn a language from data with ML/statistics — better at context ("This camera sucks" vs. "This vacuum really sucks"), negations, sarcasm. Best-known tasks: PoS tagging, NER (PERSON/ORG/GPE… with a salience score), and topic modeling via LDA / LSA. Status: mostly replaced by deep learning today.

Word embeddings

Word embedding
A dense vector of real values per word, with dimension much smaller than the vocabulary. Each word is a point in "embedding space"; cosine or Euclidean distance defines relatedness.
Why dense beats sparse

BoW / n-grams / TF-IDF are very sparse and high-dimensional (like dummy-encoding high-cardinality categoricals) — hard for ML, and worse for sequences. We want dense, low-dimensional continuous vectors learned from data where distance equals similarity.

The canonical analogy \[ [\text{king}] - [\text{man}] + [\text{woman}] \approx [\text{queen}] \]

Embeddings capture Male–Female, verb-tense, and Country–Capital relations as roughly parallel offset vectors, enabling similarity and analogy tasks. They are learned via a self-supervision task that uses context.

word2vec: CBOW vs. skip-gram

word2vec (Google, Mikolov et al., 2013) is a neural model that learns one vector per word from context. It works at the term level, not the document level. A word is defined by its context. Two methods:

CBOW (Continuous Bag-of-Words)

  • Input = context words; predict the focus word ("word against context").
  • Hidden activation = weighted average; final layer = softmax.
  • Context order does NOT matter (assumes input independence).
  • Averaging gives smoothing → less training data needed.

Skip-gram (CSG)

  • Input = focus word; predict the context ("context against word").
  • Hidden = pass-through (no activation).
  • Easier to implement.
  • Better-quality vectors but needs MORE data.
After training

Throw the network away and keep only the hidden-layer weights \(W\); the hidden-layer size = word-vector dimensionality. Comparable to an auto-encoder: dimensionality reduction with context. Good implementations use subsampling (skip frequent words) and negative sampling (don't update all weights). Visualize 1000-D vectors via a second PCA / t-SNE step.

V=10, N=4, C=3. The three one-hot contexts select weight rows [0,0,1,0], [1,0,0,-1], [1,1,0,0]. Column means → averaged activation:

\[ \tfrac{1}{3}\big([0,0,1,0]+[1,0,0,-1]+[1,1,0,0]\big) = [0.66,\ 0.33,\ 0.33,\ -0.33] \]

⚠ Exam watch-out

GloVe argues word2vec is suboptimal because it does not fully exploit global co-occurrence statistics — GloVe fixes that. fastText can vectorize any word, even made-up or misspelled ones, because its vectors are built from character substrings. word2vec and GloVe cannot. fastText is still used in industry.

Beyond words: doc2vec, entity embeddings, generalization

  • doc2vec / par2vec: adds a paragraph-id vector alongside word vectors → an embedding for a whole document.
  • Entity / categorical embeddings: embed high-cardinality categoricals to reduce dimensionality and overfitting. Jeremy Howard's size heuristic: \( 1.6 \cdot \lvert\text{cats}\rvert^{0.56} \) (one-hot day-of-week needs 6 dims; embeddings shrink this).
  • Generalization: an embedding is just a layer mapping a categorical input to a vector of trainable weights. t-SNE, UMAP, PCA, and auto-encoders are all ways to build representations. node2vec extends this to graph nodes; WEAT reveals embedding bias.

Recurrent neural networks (RNN)

RNN
Inputs are fed in sequentially; the net keeps a state = a compact summary of everything from the past needed to solve the task. "Recurrent" = it performs the same task for every element, with output dependent on previous computations.

An RNN cell with a self-loop equals an unrolled chain A→A→A→… across time — a deep network with weight sharing between time steps. Training is Backpropagation Through Time (BPTT): backprop on the time-unfolded graph because the state depends on the same weight at every past step.

Plain RNN cell — output equals the hidden state \[ a^{\langle t\rangle} = \tanh\!\big(W_{ax}\,x^{\langle t\rangle} + W_{aa}\,a^{\langle t-1\rangle} + b_a\big) \qquad \hat{y}^{\langle t\rangle} = \mathrm{softmax}\!\big(W_{ya}\,a^{\langle t\rangle} + b_y\big) \]
RNN training is hard

Vanishing gradients return; the initial-state impact can be severe; it is a nonlinear dynamical system prone to bifurcations (instability). In theory RNNs handle arbitrarily long sequences, but in practice they look back only a few steps.

Both use gating — conditionally setting / resetting state values — so the net can learn what to remember and what to discard over an undetermined time (e.g. "a bracket is still open"). GRUs are used at least as often as LSTMs now; try both.

Bidirectional RNN & seq2seq

A bidirectional RNN runs one RNN forward and one backward, combining states. This is NOT cheating by looking at the future — the full sequence is already available for offline tasks like tagging.

seq2seq
Encoder (LSTM/GRU) compresses the whole input into a fixed-length context vector; decoder (LSTM/GRU) is initialized with that context vector and generates the output sequence (e.g. translation). Context-vector length = encoder units (256/512/1024). Long sentences stay problematic because everything is squeezed into one vector.

Attention

Why attention

It is unreasonable to encode a long sequence into one fixed state vector. (A cheap trick: reverse the source sequence — it shortens the decoder-to-encoder path and helps.) Attention lets the decoder learn what to focus on while generating each output token.

Two differences from plain seq2seq: (1) the encoder passes ALL hidden states (not only the last); (2) before each output the decoder scores each encoder state, softmaxes the scores, and sums the score-weighted states into a per-step context vector.

  1. Prepare encoder hidden states \(h_1,h_2,h_3\) and the current decoder state.
  2. Score each hidden state (e.g. 13, 9, 9).
  3. Softmax the scores (e.g. 0.96, 0.02, 0.02).
  4. Multiply each state vector by its softmaxed score.
  5. Sum → context vector for this decoder step.
Query / Key / Value framing — scaled dot-product (Vaswani 2017) \[ \text{attn} = \mathrm{softmax}\!\left( \frac{Q K^{\top}}{\sqrt{d_k}} \right) , \qquad \text{output} = \text{attn} \times V \]

Query = "what do I want to look up?" (decoder state). Keys & Values = the encoder hidden states. With attention we no longer compress the whole source into one fixed vector — the cornerstone idea that led to: "maybe we can do away with recurrence altogether?"

The Transformer

Transformer
Vaswani et al., 2017, "Attention is all you need." A stack of encoders + decoders built on self-attention — Q, K, V all come from the same set of tokens. GPT and friends use a decoder-only Transformer.

Self-attention

Each decoder block has two parts: self-attention + a per-token dense layer, plus residual (skip) connections. Self-attention asks per token: "what other tokens should I look at, and how important is each?" It gradually refines each token's representation by letting it repeatedly look at the others.

Self-attention in matrices \[ Q = X W^{Q},\quad K = X W^{K},\quad V = X W^{V} \qquad Z = \mathrm{softmax}\!\left( \frac{Q K^{\top}}{\sqrt{d_k}} \right) V \]
⚠ Exam watch-out — masking & the two sublayers

The attention matrix is masked so each token attends only to previous tokens — "this is why GPT can generate text." And the bold contrast: attention mixes information BETWEEN tokens; the feed-forward network processes each token INDEPENDENTLY. (A classic true/false swap.)

Multi-head attention & positional encoding

  • Multi-head attention: not one but multiple sets of Q/K/V weight matrices ("representation subspaces"), letting the model focus on different positions. Concatenate all heads \(Z_0\ldots Z_7\), then multiply by \(W^O\) → final \(Z\) sent to the FFNN.
  • Positional encoding: self-attention has no notion of word order, so a sinusoidal positional vector is ADDED to each input embedding.
Output head — same linear layer at every position \[ \text{logits} = X \cdot W_{\text{vocab}} \]

During training all positions are scored in parallel; at inference only the last position is used, its prediction appended, and the model run again — autoregressive generation.

GPT & RLHF

GPT = Generative Pretrained Transformer: bigger decoder-only models trained on text completion given a prompt, using Byte Pair Encoding (BPE) tokenization and masked (causal) self-attention. Across generations they added more data, tool calling, reasoning tokens, multi-modality, mixture-of-experts — but the base architecture stayed the same.

RLHF — Reinforcement Learning from Human Feedback

A good loss for "desirable text" is intractable, so LLMs train on next-token-prediction; metrics like BLEU / ROUGE approximate human preference. RLHF uses human feedback as the measure / loss to align a general model with human values. Already used on top of GPT-3 (InstructGPT) and part of ChatGPT.

Beyond Transformers

The Transformer bottleneck

To generate the next token a Transformer must attend over all previous tokens → roughly \(L^2\) computations over a full sequence of length \(L\) (even with KV caching). An RNN only needs the previous hidden state + current input, so its inference scales linearly and (in theory) has unbounded context — but RNNs forget and are slow to train (BPTT).

  • State Space Models (SSM): a "state" = the minimum set of variables describing the system; SSMs predict the next state from input. Mamba (Gu & Dao, 2023, "linear-time sequence modeling with selective state spaces"); xLSTM (2024).
  • Diffusion LLMs: start from corrupted / random tokens and gradually denoise into text. Non-autoregressive → parallel token prediction, but quality has historically lagged Transformers.
  • Agents: "an LLM agent runs tools in a loop to achieve a goal."

Why is RNN inference cheaper than Transformer inference for very long generated sequences?

  1. RNNs are faster to train thanks to BPTT, which carries over to inference.
  2. RNNs attend to all previous tokens at once, avoiding recomputation.
  3. An RNN only needs the previous hidden state + current input, so inference scales linearly, whereas a Transformer recomputes attention over the whole sequence (≈ \(L^2\)).

⏱️ One-minute recap

  • TF divides by document length \(\lvert d\rvert\); IDF = \(1+\log(\lvert c\rvert/\mathrm{df})\), large for rare terms. BM25 saturates TF and can beat embeddings.
  • CBOW: context→word, averaging activation, order-independent, less data. Skip-gram: word→context, pass-through, better vectors, more data.
  • Keep only the hidden weights \(W\) after training; size = vector dim. GloVe = global co-occurrence; fastText = subword → handles OOV words.
  • RNN output = hidden state; trained with BPTT. LSTM passes two states (\(C_t\)+\(h_t\)); GRU passes one.
  • Plain seq2seq passes only the last encoder state; attention passes all, scoring → softmax → weighted sum. Scaled dot-product: \(\mathrm{softmax}(Q^{\top}K/\sqrt{n})\,V\).
  • Transformer = self-attention (Q,K,V same set). Attention mixes between tokens; FFNN is per-token. GPT = decoder-only + masked self-attention; positional encoding is added. RLHF aligns via human feedback.
  • Transformer generation ≈ \(L^2\); RNN / SSM / diffusion are the cheaper-scaling alternatives.
