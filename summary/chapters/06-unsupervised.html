Unsupervised Learning

Clustering, dimensionality reduction, anomaly detection and association rules — the high-yield distinctions, formulas and traps, distilled for a fast revision pass.

Big picture: supervised vs unsupervised

Unsupervised learning describes patterns in data — no labels required. The four families in this chapter:

  • Clustering — group objects so within-group similarity is high, between-group similarity low.
  • Dimensionality reduction — project high-dim data to a lower-dim space (PCA, t-SNE, UMAP).
  • Anomaly detection — flag instances that do not conform to the expected pattern.
  • Association rules — discover "interesting" relations between items (market-basket, apriori).

Supervised

Labelled data; correlate features to a target. Usually predictive. Split: classification (categorical target) vs regression (continuous target).

Unsupervised

No labelling needed; clustering, association/sequence rules. Usually descriptive / explanatory.

◆ Key framing quote

"For unsupervised learning, we don't assume a label or target … but we do need to define what kind of patterns we want." No target, but you still choose what "similar" / "interesting" means.

Clustering: definition & two types

Clustering
Grouping objects so members of a group are more similar to each other than to members of other groups. A main task of exploratory data mining.
◆ Why cluster?

Reveals the internal structure of data. Sometimes the partition is the goal itself (market segmentation). It is also a first step toward predictive techniques — mine a decision tree on cluster labels, or use labels / distances as features.

Hierarchical vs partitional

Hierarchical

Nested decomposition (connectivity / distance based) → a dendrogram.

  • Agglomerative = bottom-up (singletons → merge into clusters).
  • Divisive = top-down (whole set → divide into partitions).

Partitional

Objective-function based: build partitions, score them. Each instance in exactly one of K non-overlapping clusters. Examples: K-means, K-means++.

⚠ Exam watch-out

Divisive = top-down (whole set splits to singletons); Agglomerative = bottom-up (singletons merge to one cluster). Don't swap them. All real implementations are agglomerative — divisive is way more expensive (it must consider every possible split).

Hierarchical clustering mechanism

To decide which clusters to merge you need a (dis)similarity between sets of observations, built from two ingredients:

  1. A distance metric — distance between pairs of observations.
  2. A linkage criterion — dissimilarity of sets as a function of those pairwise distances.

Distance metrics (point-to-point)

Euclidean (L2)\[ \lVert a-b\rVert_2 = \sqrt{\textstyle\sum_i (a_i-b_i)^2} \]
Squared Euclidean (used in SSE)\[ \lVert a-b\rVert_2^2 = \sum_i (a_i-b_i)^2 \]
Manhattan (L1) — axis-aligned\[ \lVert a-b\rVert_1 = \sum_i \lvert a_i-b_i\rvert \]
Maximum / Chebyshev (L∞)\[ \lVert a-b\rVert_\infty = \max_i \lvert a_i-b_i\rvert \]
◆ A valid metric must satisfy 4 properties

Symmetry \( d(a,b)=d(b,a) \); self-similarity = 0 \( d(a,a)=0 \); positivity \( d(a,b)\ge 0 \); triangle inequality \( d(a,b)\le d(a,c)+d(c,b) \).

Linkage criteria (set-to-set)

LinkageDefinitionEffect
Single (min)\( \min d(a,b) \)Longer, skinnier clusters (chaining)
Complete (max)\( \max d(a,b) \)Tight / compact clusters
Average (mean)\( \frac{1}{|A||B|}\sum_{a,b} d(a,b) \)Favorable, robust to noise
Centroid\( d(a_c, b_c) \)Robust; needs a centroid concept
Ward's\( \mathrm{SSE}(AB) - (\mathrm{SSE}(A)+\mathrm{SSE}(B)) \)Increase in SSE on merge; ANOVA-like
⚠ Exam watch-out

Don't swap single ↔ complete: single → chaining / skinny, complete → tight. Ward's = increase in error sum of squares after merging (distance-based SSE, not target-vs-prediction). Hierarchical clustering needs no k in advance but requires normalization, is not very scalable, and can hit local optima.

Dendrogram

  • A "cut" across the dendrogram decides the number of clusters.
  • Instances that join very late / at large heights are good candidate outliers / anomalies.

K-means

K-means
Non-hierarchical, flat partition into K clusters; user supplies k. Iterative assign-and-update heuristic.
  1. Decide k.
  2. Initialize k cluster centers (random in feature space or random instances).
  3. Assign each instance to its nearest center.
  4. Recalculate the k centers.
  5. Repeat 3–4 until no instance changes cluster.
◆ Objective

A good solution has high intra-cluster and low inter-cluster similarity. K-means minimizes total distortion (sum of squared distances to the centroid):

Total distortion / SSE\[ \min \;\mathrm{SSE} = \sum_{k=1}^{K}\sum_{i=1}^{n_k} \lVert x_{ki} - \mu_k \rVert^2 \]

\( \mu_k \) = centroid of cluster k, \( n_k \) = points in cluster k.

◆ Heuristic, not exact

Exact optimization of this objective is hard. K-means is a heuristic — not guaranteed to find the global optimum, except in the one-dimensional case.

Strengths & weaknesses

Strengths

  • Simple to implement / debug.
  • Intuitive objective (intra-cluster similarity).
  • Relatively efficient.

Weaknesses

  • Needs the mean defined → struggles with categorical data.
  • Local optima → init matters (random restarts / K-means++).
  • Must specify k (use the elbow).
  • Sensitive to noise/outliers; often needs normalization.
  • Not suitable for non-convex shapes or differing densities.

K-means++ (smart initialization)

  1. Pick the first center uniformly at random.
  2. For each point x, compute \( d(x,c) \) = distance to the nearest already-chosen center.
  3. Choose the next center with probability proportional to \( d(x,c)^2 \).
  4. Repeat until k centers, then run standard k-means.
✓ Idea

Spread initial centers apart → lower inter-cluster similarity at start, fewer bad local optima. Most k-means libraries use K-means++ by default.

1

In K-means++, after the first center is chosen at random, each subsequent center is selected with probability proportional to what?

  1. Uniformly at random over all points
  2. \( d(x,c)^2 \), the squared distance to the nearest existing center
  3. \( 1/d(x,c) \), the inverse distance to the nearest center

Density & model-based clustering

DBSCAN

DBSCAN
Density-Based Spatial Clustering of Applications with Noise. Groups densely packed points; marks isolated low-density points as noise/outliers. Parameters: Eps (radius) and MinPts.

EM / Gaussian mixtures

EM clustering (GMM)
Clustering via a mixture of Gaussians. E-step: assign probabilistic (soft) membership. M-step: re-estimate model parameters.
◆ EM vs K-means

K-means makes hard assignments; EM makes soft (probabilistic) ones — "a less hard version of k-means." EM learns a model per cluster (so it can generate new points) and extends to other model types (multinomial for categorical). But it still needs k, still has local optima, still struggles with non-convex shapes.

Mean-shift

Built on kernel density estimation: place a kernel (usually Gaussian) on each point, sum them into a density surface, then shift points toward the modes (peaks). Key parameter = bandwidth. Does not require specifying k.

Which statement about clustering algorithms is true?

  2. DBSCAN requires you to specify the number of clusters k

Validating clustering

Use the k-means objective (SSE), or indices such as the Davies–Bouldin Index (DBI):

Davies–Bouldin Index\[ \mathrm{DBI} = \frac{1}{N}\sum_{i=1}^{N} R_i,\qquad R_i = \max_{j\neq i} \frac{S_i + S_j}{D_{ij}} \]

\( S_i \) = within-cluster spread (mean distance to centroid); \( D_{ij} \) = distance between centroids i and j.

⚠ Exam watch-out

DBI: LOWER is better — low within-cluster spread \( S_i \) and high between-cluster separation \( D_{ij} \). Validation here is somewhat subjective given the descriptive nature.

Text & mixed distance metrics

  • Levenshtein (edit) distance: number of deletions/insertions/substitutions to turn one string into another.
  • Jaccard, cosine (text/sets); Gower distance handles mixed categorical + numerical data.

Domain knowledge enters clustering via must-link / can't-link constraints (too many → infeasible) or a warm start from a partial solution.

Dimensionality reduction: PCA

PCA
Linear DR. Principal components from the eigenvector decomposition of the covariance matrix. The eigenvalue \( \lambda_j \) = explained variance of component j.
Principal component j\[ \mathrm{PC}_j = e_j' X = e_{j1}X_1 + e_{j2}X_2 + \dots + e_{jp}X_p \]
  • Pro: powerful reduction; PCs are uncorrelated.
  • Con: hard to interpret; linear only (misses non-linear manifolds).

t-SNE

t-SNE
Non-linear manifold DR that preserves local similarities (small pairwise distances).
Manifold
A space that locally resembles Euclidean space near each point.

How it works — two steps

Step 1 — high-dim

Gaussian kernel centered at each point gives conditional probabilities, then symmetrize into joint \( p_{ij} \). Uses perplexity to set \( \sigma_i \).

Step 2 — low-dim map

Student t-distribution (fatter tails) gives \( q_{ij} \) between mapped points. No perplexity parameter here.

Conditional similarity (Gaussian)\[ p_{j\mid i} = \frac{\exp\!\left(-\lVert x_i - x_j\rVert^2 / 2\sigma_i^2\right)}{\sum_{k\neq i}\exp\!\left(-\lVert x_i - x_k\rVert^2 / 2\sigma_i^2\right)} \]
Symmetrized joint\[ p_{ij} = \frac{p_{j\mid i} + p_{i\mid j}}{2N},\qquad p_{ii}=0 \]
Optimize — minimize KL divergence (gradient descent)\[ \mathrm{KL}(P\Vert Q) = \sum_{i\neq j} p_{ij}\,\log\frac{p_{ij}}{q_{ij}} \]
◆ Perplexity (key hyperparameter)

≈ number of effective nearest neighbors (like k in k-NN). It sets the data-point-dependent bandwidth \( \sigma_i \) — smaller \( \sigma \) in denser regions. Typical values 5–50; a denser dataset needs larger perplexity.

⚠ Exam watch-out

t-SNE is NOT a clustering technique — it's DR (you can cluster its output, carefully). In a t-SNE plot, cluster SIZE and inter-cluster DISTANCE are NOT meaningful, and random data can look structured. t-SNE is non-parametric → cannot embed unseen test points (poor for predictive featurization; parametric t-SNE fixes this). Step 1 = Gaussian, Step 2 = Student-t; loss = KL divergence.

UMAP

Like t-SNE, a non-linear manifold DR — used as a drop-in t-SNE replacement. Advantages:

  • Parametric by default → can embed new points → good general-purpose preprocessing.
  • Can incorporate supervised labels in the mapping.
  • Scales better than t-SNE and MDS; better for dims greater than 3.
  • Nicer interpretation properties than t-SNE.

Anomaly detection

Anomaly / outlier / novelty detection
Identifying instances that do not conform to an expected pattern (fraud, defects, errors). Mixes unsupervised, supervised and statistical methods.

⚠ Exam watch-out

Isolation forest: splits chosen completely at random (NOT entropy-driven); anomalies get short root-to-leaf paths (separated quickly). LOF is k-NN based — flags points with substantially lower local density than their neighbors. (V)AE reconstruction error is a proxy for anomaly score. CADE: real instances labeled y=0, synthetic uniform-generated instances labeled y=1; classifier scores originals.

Association rules

Association rule learning
Discovering "interesting" relations between items using measures of interestingness. Each instance = a transaction; features = items (binary: 1 = present). Order is ignored (sequence mining respects order). Pioneering technique: apriori.

Support, confidence, lift, conviction

Support of itemset X\[ \mathrm{Support}(X) = \frac{|\{t \in T : X \subseteq t\}|}{|T|} \]
Confidence of X ⇒ Y (estimate of P(Y|X))\[ \mathrm{Confidence}(X\Rightarrow Y) = \frac{\mathrm{Support}(X\cup Y)}{\mathrm{Support}(X)} \]
Lift\[ \mathrm{Lift}(X\Rightarrow Y) = \frac{\mathrm{Support}(X\cup Y)}{\mathrm{Support}(X)\cdot \mathrm{Support}(Y)} \]
Conviction\[ \mathrm{Conviction}(X\Rightarrow Y) = \frac{1-\mathrm{Support}(Y)}{1-\mathrm{Confidence}(X\Rightarrow Y)} \]
◆ Reading the measures

Confidence ≈ \( P(Y\mid X) \) — divide by Support(X), not Support(Y). Lift = 1 → independence; Lift greater than 1 → positive dependence. Lift considers both the rule's confidence and the whole dataset. Conviction = ratio of expected wrong predictions under independence.

Worked numbers (5-transaction DB)

Support({milk, bread, cheese}) = 2/5 = 0.4 (only transactions 103 & 105 contain all three).

Confidence({cheese, wine} → {spaghetti}) = Support({cheese,wine,spaghetti}) / Support({cheese,wine}) = \( 0.4 / 0.6 = \mathbf{0.66} \).

4

A rule \( X\Rightarrow Y \) has \( \mathrm{Lift} = 1 \). What does this mean?

  1. X and Y are statistically independent
  2. X and Y are strongly positively associated
  3. The rule has the maximum possible confidence

The apriori algorithm

  1. Apply a minimum support threshold → all frequent itemsets.
  2. Apply a minimum confidence threshold to those → association rules.
◆ Why apriori needs a trick

The set of possible itemsets is the power set over the items, of size \( 2^{|I|}-1 \) (excluding the empty set). For 100 items that's \( 2^{100}-1 \) — brute force is infeasible (even 6 items = 63 itemsets).

◆ Downward-closure / anti-monotonicity

Every subset of a frequent itemset is frequent ⇒ every superset of an infrequent itemset is infrequent. This pruning rule lets apriori avoid the whole power set.

Join-and-prune for k-itemsets: join the frequent (k−1)-itemsets with themselves to form length-k candidates, then prune any candidate containing an infrequent (k−1)-subset. Rule generation: for each frequent itemset I, test all non-empty proper subsets \( I_s \) and keep \( I_s \Rightarrow I\setminus I_s \) when confidence exceeds minconf.

⚠ Exam watch-out

Apriori order: (1) min support → frequent itemsets, then (2) min confidence → rules. Power-set size = \( 2^{|I|}-1 \). Confidence is not always the best metric — use lift/conviction/rarity. Negatively-correlated / rare patterns can be more interesting than frequent ones; sequence mining (GSP) keeps order, plain apriori ignores it.

Practical mining notes

  • Categorical/continuous: dummy-code levels into binary "items"; bin continuous variables (needs fine-tuning) and drop overly frequent levels.
  • Post-processing is a necessity: lots of rules appear → sensitivity analysis on minsup/minconf, drop trivial rules, hunt unexpected/actionable ones. Visualization (R arulesViz) is crucial.

One-minute recap

  • Unsupervised = describe patterns, no labels — but you still define the pattern you want.
  • Agglomerative = bottom-up (all implementations); divisive = top-down (expensive).
  • Single linkage → chaining/skinny; complete → tight; Ward's = SSE increase on merge.
  • K-means minimizes \( \sum\lVert x-\mu_k\rVert^2 \); heuristic (global only in 1D); needs k, mean, normalization; fails on non-convex. K-means++ init ∝ \( d(x,c)^2 \).
  • DBSCAN (Eps, MinPts) → Core/Border/Noise, handles non-convex. EM/GMM = soft k-means. Mean-shift = KDE, bandwidth, no k.
  • DBI: lower is better.
  • PCA linear, uncorrelated PCs, \( \lambda_j \) = explained variance. t-SNE non-linear, local similarities, perplexity 5–50, Gaussian → Student-t, minimizes KL; size/distance not meaningful, non-parametric. UMAP = parametric, label-aware, scalable t-SNE replacement.
  • Anomaly: isolation forest (random splits, short paths), LOF (k-NN density), (V)AE (reconstruction error), CADE (real y=0 vs synthetic uniform y=1).
  • Confidence = Support(X∪Y)/Support(X) ≈ P(Y|X). Lift=1 independent. Apriori: support then confidence; anti-monotonicity prunes; \( 2^{|I|}-1 \) itemsets.
