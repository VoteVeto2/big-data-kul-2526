Big picture: why ensembles

A single very deep tree overfits (train accuracy keeps climbing while test accuracy peaks then drops); a single simple model underfits. Pruning / early stopping buy robustness only by sacrificing peak accuracy. Ensembles let you "have it all": combine many base models to either reduce variance (bagging) or reduce bias (boosting) while keeping good accuracy.

◆ Why combine models

Two models can each be best on a different subsegment of the data; combining them lifts performance and adds robustness to noise/overfitting. The "wisdom of the crowd" jelly-bean-jar: many noisy individual guesses average out near the truth (mean \( \approx 487 \) vs. the true count).

The price ensembles pay: interpretability and production simplicity — "how do you explain 100 trees vs. 1?"

Combining model outputs

Combination rules (algebraic)
Combine probability outputs by Min / Max / Mean, then find a new optimal cutoff — \( 0.50 \) is no longer valid. No rule is universally best.
Voting
Combine discrete decisions; majority wins (or weighted votes). Advantage: less sensitive to the probability distributions — no calibration / new cutoff needed.
Mixture of experts (Jacobs/Jordan, 1991)
Experts \( C_1\dots C_T \) combined linearly with weights from a gating network (trained by EM): \( H=\sum_i w_i\, h_i(x,\theta_i) \).
Stacking (Wolpert, 1992)
Tier-1 base models feed their outputs into a Tier-2 meta-classifier that learns whether the data were properly learned and corrects base errors.
Smoothing
Convex blend of a simple and a complex model: \( \text{Ensemble} = \lambda\cdot(\text{logreg}) + (1-\lambda)\cdot(\text{complex}) \).
⚠ Exam watch-out

After any algebraic combination (Min/Max/Mean) you must re-tune the cutoff. Worked slide: Min→\(0.50\), Max→\(0.78\), Mean→\(0.52\); only Mean gets all 6 instances right, while Max wrongly labels a true "Yes" (\(0.54\)) as "No". Mixture of experts ↔ gating network + EM (1991); stacking ↔ Tier-2 meta-classifier (1992) — don't swap them.

Bagging (Bootstrap Aggregating)

Bagging = train many classifiers of the same type, each on a bootstrap replica (sample with replacement) of the training data, then combine by majority vote. Core idea: add diversity; diverse models make different errors that cancel out → lower variance.

  1. Draw bootstrap samples (random, with replacement) from the training set.
  2. Train one same-type classifier per sample (use unstable learners like trees, whose output changes a lot with small data changes).
  3. Combine via majority vote (original publication) — or average probabilities (scikit-learn).
Bagging ensemble outputs \[ p_{\text{ens}}(y{=}1\mid x)=\frac{\big|\{c\in C : p_c(y{=}1\mid x)\ge 0.5\}\big|}{|C|} \qquad\text{or}\qquad \frac{1}{|C|}\sum_{c\in C} p_c(y{=}1\mid x) \]
◆ scikit-learn detail

scikit-learn combines by averaging probabilistic predictions, not by letting each classifier vote a single class (contrast with the original Breiman publication). For large ensembles the two often agree, but are not guaranteed to — they can diverge when the base probabilities are poorly calibrated or cluster near 0.5.

Out-of-bag (OOB) validation

A free generalization estimate: each bootstrap leaves out \( \approx 37\% \) of instances, so each instance is unseen by some models. Validate instance \(i\) using only the models whose bootstrap excluded \(i\).

⚠ Exam watch-out

OOB is a good initial check but does NOT replace an independent test set. "Bootstrap size 100%" does not mean all instances — drawing \(N\) times with replacement leaves out \( e^{-1}\approx 0.368 \) of them (the OOB set).

Random forests

Random forest = bagged decision trees + random feature subset at each split. Two sources of randomness control variance by design — so no pruning is needed (pruning is only an optional speed-up).

⚠ Exam watch-out — the classic trap

The random feature subset is sampled at EVERY split, not once per tree. Feature-subset size heuristic: \( M=\lfloor N/3\rfloor \) (min 1) for regression, \( M=\lfloor\sqrt{N}\rfloor \) for classification (don't swap them; e.g. \(N=16\) → 5 vs. 4). Adding more trees cannot overfit a forest — validation accuracy stays flat.

Extra-Trees add a third randomization: a random subset of split points, not just features.

Boosting (general idea)

Boosting builds classifiers sequentially, without bootstrapping; each new model corrects the mistakes of the ensemble so far. Core idea: cooperation (not diversity) → reduces bias. Misclassified instances get heavier weight; the next model focuses on them. Risk: overboosting (overfitting from too many rounds).

AdaBoost (Freund & Schapire, 1996)

Labels \( y\in\{-1,+1\} \). Trains weak learners over \(T\) rounds, reweighting instances each round and weighting each learner by its accuracy.

D_1(i) = 1/N                          # uniform weights
for t = 1..T:
    h_t = train weak learner on (S, D_t)
    eps_t = sum of D_t(i) over misclassified i
    if eps_t > 0.5:  T = t-1; STOP    # worse than random -> stop
    alpha_t = 0.5 * ln((1 - eps_t)/eps_t)
    D_{t+1}(i) = D_t(i) * exp(-alpha_t * h_t(x_i) * y_i)
    normalize D_{t+1} to sum to 1
output H(x) = sign( sum_t alpha_t * h_t(x) )
Learner weight \[ \alpha_t=\tfrac{1}{2}\ln\!\frac{1-\varepsilon_t}{\varepsilon_t} \qquad \varepsilon_t=0.25\Rightarrow\alpha_t=\tfrac12\ln 3\approx 0.549,\quad \varepsilon_t=0.5\Rightarrow\alpha_t=0 \]
⚠ Exam watch-out

AdaBoost stops if \( \varepsilon_t \gt 0.5 \) (learner worse than random). Low \( \varepsilon_t \) → high \( \alpha_t \) → bigger vote. Friedman et al.: AdaBoost is an additive logistic-regression model minimizing an EXPONENTIAL loss. It is sensitive to noise/outliers (they get up-weighted repeatedly).

Gradient Boosting Machines (GBM)

Fits each sequential learner (a weak tree) to the negative gradient of the loss w.r.t. the model's raw output — i.e. functional gradient descent, so any differentiable loss can be optimized. For squared error, the negative gradient equals the residual:

Residual = negative MSE gradient \[ L_{\text{MSE}}=\tfrac1n\big(y-F(x)\big)^2 \;\Rightarrow\; -\frac{\partial L_{\text{MSE}}}{\partial F}=\tfrac{2}{n}\big(y-F(x)\big)=\tfrac{2}{n}\,h_m(x) \]
◆ General statement

GBM fits the negative gradient of the loss in general; residual-fitting is just the special case for MSE. Swap in another loss + its gradient: LogLoss (binary), MLogLoss (multiclass). Quirk (classification-margin view, \(y\in\{-1,+1\}\)): squared-error loss \( (y-F)^2 \) rises again once \( yF\gt 1 \), penalizing predictions that are "too correct" — this does not apply to ordinary regression MSE.

XGBoost / LightGBM / CatBoost

Extreme gradient boosting improves the optimization method and adds regularization. Two key improvements:

  • Second-order optimization: uses the gradient and the Hessian (Jacobian + Hessian) → reaches the loss minimum faster. Plain GBM uses first-order gradients only.
  • Regularization (L1/L2): objective = loss + simplicity of the trees (over depth / number of leaves) → better generalization.

Refinements: DART, interaction & monotonic constraints

DART (2015)
Applies neural-net dropout to boosted trees. Motivation: with many trees + small learning rate, early trees dominate and late trees become unimportant; dropping trees mitigates this over-specialization.
Feature interaction constraints
Variables on the same traversal path interact (a child's condition is predicated on its parent). Depth > 1 can capture spurious (noise) interactions; constraints restrict which features may interact.
Monotonic constraints
Force a feature's effect on the prediction to be monotonic (e.g. risk only increases with a feature) — encodes domain knowledge and smooths noisy wiggles into generalizing trends.

⚠ Exam watch-out — favorite trap

Bagging reduces variance; boosting reduces bias. Reversing these is the #1 trap. Empirically: more trees in a random forest keep validation flat (robust); more AdaBoost rounds push training up but validation peaks then declines (overboosting). Boosting uses no bootstrap — it reweights instances.

Deep learning for tabular data

Can deep nets beat tree ensembles on tabular data? "In most cases: no(t really)." Tabular data has no spatial structure (unlike CNNs), no sequential structure (unlike NLP), is often small (10k–100k rows), and tree ensembles capture nonlinear interactions for free.

  • MLPs — strong, simple baseline (numeric normalized, categorical embedded/one-hot) but underperform GBMs.
  • Tabular-specific nets (TabNet, FT-Transformer, SAINT, NODE/DeepGBM) — treat each feature as a "token" and use self-attention to learn interactions.
  • Gradient boosting still wins honest benchmarks.

TabPFN — Prior-Data Fitted Networks

TabPFN (Tabular Prior-Data Fitted Network)
A foundation-model approach: a Transformer trained once on millions of synthetic tabular tasks, then applied directly to new datasets. At inference you feed your small dataset in-context — no per-dataset training, no hyperparameter tuning. It learns Bayesian inference over tasks → approximates Bayesian model averaging.
◆ Status

v1 underperformed; v2 / v2.5 are often on par with or better than GBMs — the emerging exception to "trees win". Datasets > 10k rows are trickier but feasible.

One-minute recap

  • Bagging = same-type models on bootstrap replicas, combine by vote → reduces variance; diversity; parallel. OOB ≈ 37% left out (not a test-set replacement).
  • Random forest = bagging + random feature subset at every split (\(\lfloor\sqrt N\rfloor\) classif., \(\lfloor N/3\rfloor\) regr.); no pruning needed; more trees never overfit.
  • Boosting = sequential, no bootstrap, reweight mistakes → reduces bias; cooperation; can overboost.
  • AdaBoost: \( y\in\{-1,+1\} \), \( \alpha_t=\tfrac12\ln\frac{1-\varepsilon_t}{\varepsilon_t} \), stop if \( \varepsilon_t\gt0.5 \); minimizes exponential loss.
  • GBM: fits negative gradient of loss (residual = MSE special case). XGBoost: grad+Hessian + L1/L2. Growth: XGB level-wise, LightGBM leaf-wise, CatBoost symmetric.
  • Tabular: gradient boosting usually beats deep learning; TabPFN v2/v2.5 (in-context, no tuning, ≈ Bayesian model averaging) is the emerging exception.
